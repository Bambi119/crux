# 5. 전차 시스템

> 개정: 2026-04-15 — 부대 단위 파츠 경제·호환성 3중 체계·5 차체 슬롯 구조로 전면 재작성
> 이전 판본(Front Mission식 단일 전차 커스터마이징)은 단일 전차 가정이었고, 현재는 **부대 N대 공용 인벤토리** 기반.

## 설계 철학

1. **파츠 조합이 플레이 스타일을 결정** — 동일 차체라도 엔진·주포·장갑 조합에 따라 완전히 다른 역할
2. **부대 공용 인벤토리** — 파츠는 전차가 아닌 **부대**에 귀속. 전차 간 자유 이동
3. **호환성 3중 체계** — 단순 슬롯 매칭이 아니라 하중·출력·규격 세 축 전부 통과해야 조립 성립
4. **노획 = 핵심 경제 사이클** — 전투에서 **어떻게 격파했느냐**가 곧 다음 빌드 재료를 결정
5. **로시난테 특례** — 중형 차체 고정, 서사적 전차. 다른 모든 전차는 대체·노획 가능

## 1. 차체(Hull) 5종

차체는 **부대의 골격**. 변경 불가. 파츠 슬롯 수·하중 한도·출력 요구가 차체 종속.

| 차체 | 중량대 | 슬롯 수 | 특징 | 역할 |
|---|---|---|---|---|
| **Scout** (경전차) | 경 | 2 | 고속·경장·소구경 | 정찰·측면 기동·대 드론 |
| **Assault** (중형전차) | 중 | 3 | 범용 밸런스 | 주전투 라인, 로시난테 기본값 |
| **Support** (지원전차) | 중 | 4~5 | 수리·보조 장비 슬롯 풍부 | 수리·보급·제압 사격 |
| **Heavy** (중전차) | 중 | 5 | 대구경·후장 | 관통 우위·요새화 |
| **Siege** (초중전차) | 초중 | 5+ | 다연장·고정형 | 거점 공격·대보스 |

### 1.1 슬롯 구조

각 차체는 고정된 **파츠 슬롯 테이블**을 가진다. 슬롯은 **종별 카테고리**로 나뉜다:

| 슬롯 카테고리 | Scout | Assault | Support | Heavy | Siege |
|---|---|---|---|---|---|
| 엔진 | 1 | 1 | 1 | 1 | 1 |
| 포탑 | 1 | 1 | 1 | 1 | 1 |
| 주포 | 1 | 1 | 1 | 1 | 1~2 |
| 장갑(면별) | 3 | 4 | 4 | 6 | 6 |
| 탄약고 | 1 | 1 | 1 | 1 | 2 |
| 캐터필러 | 1 | 1 | 1 | 1 | 1 |
| 보조 장비 | 1 | 2 | 3~4 | 3 | 3 |

※ 장갑 슬롯 수는 **장갑면 독립 장착** 구조 — 전면·측면×2·후면×N을 별도로 관리.
※ Scout·Assault 같은 경·중 차체는 슬롯이 적어 **선택과 집중** 플레이. Heavy·Siege는 빌드 조합의 자유도 확대.

### 1.2 차체 숙련도와 조종수·전차장 선호

- **조종수 선호 중량대**와 차체 중량대가 일치 → 이동 AP -1 (docs/04 §2.2)
- **전차장 선호 전차 클래스**와 차체 클래스가 일치 → 시야 +1
- 마크 숙련도는 **차체 클래스별 독립 축** (조종수: 경·중·초중 / 전차장: 경·중·중·초중 4축)
- 같은 중형이라도 Assault와 Heavy는 각각 별도 마크 축으로 누적

### 1.3 로시난테 특례

- **중형전차(Assault) 고정**. 교체·폐기 불가
- 로시난테는 일반 Assault보다 **1 보조 장비 슬롯 추가** — 서사적 영웅 전차 보정
- 로시난테 전용 파츠 존재 (스토리 보상으로만 획득, 다른 전차에는 장착 불가)
- 상세 성장 로드맵은 본 문서 §9 참조

## 2. 파츠 카테고리

### 2.1 엔진

| 속성 | 의미 |
|---|---|
| **출력(Power)** | 전차에 공급하는 출력값. 차체별 **최소 출력 요구** 충족 필요 |
| **중량(Weight)** | 엔진 자체 무게 — 하중 한도 소모 |
| **연비** | 전략 맵 이동 거리 소모량 (현재는 전투에 영향 없음) |
| **과열 한계** | 매 턴 부스트·중량 초과 시 과열 누적. 한계 초과 시 턴 스킵 |

출력 부족 → 이동 AP 증가, 회전 AP 증가. 여유 출력 많음 → 중장갑 탑재 여유.

### 2.2 포탑

| 속성 | 의미 |
|---|---|
| **회전 속도** | 회전 AP 기준값 (조종수 마크·스킬로 보정) |
| **주포 구경 제한** | 장착 가능 주포 구경 (소·중·대) |
| **무게** | 하중 소모 |
| **고정포대 여부** | Siege 차체 일부는 포탑 대신 고정포대 슬롯 — 회전 불가 대신 안정성·중구경 허용 |

### 2.3 주포(Main Gun)

무기 스타일별 역할:

| 무기 타입 | 특성 | 역할 |
|---|---|---|
| 소구경 속사포 | 연사·낮은 관통 | 대 드론·경장갑 적 |
| 중구경 범용포 | 범용 명중·관통 | 주전투 라인 (Assault 기본값) |
| 대구경포 | 고 관통·느린 장전 | 저격형·요새 파괴 |
| 유탄발사기 | 범위 폭발 | 밀집 적·보병 |
| 로켓/미사일 | 유도·고 데미지·제한 탄약 | 보스·거점 처리 |
| 다연장 | 동시 발사·넓은 범위 | Siege 전용, 대군 처리 |

**구경과 포수 마크 축은 직결** — 대구경 전문 포수는 소구경 기관포에서 숙련 보너스 없음 (docs/04 §3).

### 2.4 탄약고

- **탄약 보유량** 결정 (탄종별 혼합 가능, 고정 슬롯 수)
- **탄종 수용** — 대부분 2~3종 제한, 대형 탄약고는 4종 가능
- **유폭 위험** — 피격 시 치명적. 특히 측·후면 피격 시 관통 확률↑ → 탄약고 유폭
- 탄약수 선호 탄종 일치 → 장전 AP -1 (docs/04 §2.2)

### 2.5 장갑

**면별 독립 장착**. 전면 / 측면 L·R / 후면의 각 슬롯에 장갑판을 개별 적용.

| 장갑 유형 | 특징 |
|---|---|
| 경장갑 | 가볍고 도탄 유도 우선 — 경사 보너스 큼 |
| 복합장갑 | 범용 — 균형잡힌 방호 |
| 중장갑 | 고 방호·매우 무거움 — Heavy/Siege만 장착 가능 |
| 리액티브 | 관통 1회 완화·파손 — 한정 재사용 |

경사(입사각) 계산은 Hex 방향 6면 기반. 상세는 docs/06.

### 2.6 캐터필러

| 속성 | 의미 |
|---|---|
| **기동성** | 기본 이동 AP 계산 |
| **내구** | 파손 확률 기준 |
| **지형 적응** | 특정 지형(습지·파편)에서 이동 AP 보너스 |

캐터필러 파손 시 이동 AP 2배. 기총사수/수리병의 **캐터필러 수리 마크**로 전투 중 복구 가능.

### 2.7 보조 장비

자유 배치 슬롯. 차체별 수량 상이 (§1.1 참조).

| 유형 | 효과 예 |
|---|---|
| 연막탄 발사기 | 1턴 LOS 차단 셀 생성 |
| 조준 보조 | 명중 +5 |
| 통신 강화 | 지휘 스킬 사거리 +2 |
| 적외선 스코프 | 연막·연기 관통 시야 |
| 응급 키트 | 수리 스킬 쿨다운 -1 |
| 부스터 | 1회 한정 이동 +3셀 |

## 3. 호환성 3중 체계

파츠 조합이 성립하려면 다음 **세 축을 모두 통과**해야 한다. 하나라도 실패 시 장착 불가 (UI 경고).

### 3.1 하중(Weight)

```
총 중량 = 차체 + 엔진 + 포탑 + 주포 + 장갑 + 탄약(최대치) + 캐터필러 + 보조 장비
하중 한도 = 차체별 고정값
```

초과 시: 장착 불가. 경계값 근접 시 과열·속도 페널티 경고.

### 3.2 출력(Power)

```
엔진 출력 ≥ 차체 최소 출력 요구 + 장착 파츠 합계 출력 수요
```

엔진이 약하면 무거운 주포나 리액티브 장갑을 돌릴 수 없다. 빌드 핵심 결정.

### 3.3 규격(Specification)

파츠 간 물리적·전기적 **규격 호환**.

| 규격 축 | 예 |
|---|---|
| 포탑 - 주포 | 포탑별 수용 구경 한계 (소·중·대) |
| 차체 - 탄약고 | Scout는 대용량 탄약고 불가 |
| 차체 - 장갑 | Scout는 중장갑 거부 |
| 엔진 - 냉각 | 고출력 엔진은 냉각 보조 장비 필요 (일부 조합) |

**규격 불일치는 하중·출력과 독립적 거부 사유.** 빌드 구성 시 UI는 각 축별 상태를 별도 플래그로 표시.

### 3.4 UI 피드백 원칙

편성 씬에서 파츠 드래그 시 세 축 상태를 실시간 표시:

```
[Pretena의 Assault #1]
하중  ▓▓▓▓▓▓▓░░░  78 / 100   ✓
출력  ▓▓▓▓▓▓▓▓▓░  92 / 100   ✓ (여유 8)
규격  ✗ 주포 '대구경 저격포' 장착 불가 — 포탑 '경량 2축' 소 구경만 수용
```

장착 불가 파츠는 슬롯 드래그 시 거부 + 사유 표기. 자동 대체 없음.

## 4. 파츠 경제

### 4.1 공용 인벤토리

- 모든 파츠는 **부대 공용 인벤토리(Hangar Stash)**에 귀속
- 전차에 장착된 파츠는 "점유 중" 마킹만 될 뿐 소유는 부대
- 전차 간 파츠 교환에 별도 비용 없음 (정비 씬에서 자유 조립)
- 동일 파츠 복수 보유 가능 (ID 기반 인스턴스)

### 4.2 파츠 획득 경로

| 경로 | 특성 |
|---|---|
| **전투 노획** | 핵심. 격파 방식이 결과 결정 (§5 참조) |
| **스토리 보상** | 거점·이벤트 클리어 시 고정 보상 |
| **분기 선택** | 스토리 경로별 상호 배타 파츠 |
| **상인·교환** | 일부 거점에서 자원 교환 (MVP 이후) |
| **로시난테 전용 보상** | 로시난테 한정 강화 — 다른 전차 장착 불가 |

### 4.3 파츠 품질 등급

| 등급 | 특성 | 획득 경로 |
|---|---|---|
| 쓰레기 | 기본 이하 성능 | 탄약고 유폭 격파·스토리 시작 |
| 일반 | 기본 성능 | 주포 직격 격파·일반 스테이지 |
| 숙련 | +10~20% 보너스 | 모듈 집중 파괴·중반 스테이지 |
| 정예 | +30~40% 보너스 | 승무원 사살 노획·후반 거점 |
| 전설 | 고유 효과 | 승무원 전원 사망 노획·스토리 이벤트 |

**등급과 격파 방식 연동**이 게임의 경제 축. 난이도 조절도 이 사다리로.

### 4.4 파츠 폐기·분해

- 부분 분해 → 해당 파츠 고유 소재로 환원
- 소재는 **같은 카테고리 내** 파츠 수리·업그레이드에 사용
- 교환 비율은 등급 하락 (정예 1개 → 숙련 재료 ~2개)

## 5. 노획 시스템

**전투에서 어떻게 격파했느냐가 곧 다음 빌드 재료를 결정**. 이것이 CRUX 경제의 핵심 피드백 루프.

> **구현 상세**: 격파 방식 판정 트리·PartCategory별 생존 확률 매트릭스·등급 변환·차체 노획 알고리즘·결과 모달 표시 규칙은 보조 문서 [`05b_salvage_algorithm.md`](05b_salvage_algorithm.md)로 분리. 본 절은 개요·설계 의도만 기술.

### 5.1 격파 방식별 노획률

| 격파 방식 | 파츠 생존률 | 차체 노획 | 비고 |
|---|---|---|---|
| 탄약고 유폭 | 5~10% | 불가 | 내부 손상 극심, 고철만 |
| 주포 관통 직사 | 20~30% | 매우 낮음 | 차체 훼손 큼 |
| 엔진 피격 파괴 | 30~40% | 낮음 | 엔진 소실 |
| 캐터필러 파손 · 무력화 | 50~60% | 중간 | 기동 불능으로 제압 후 추가 격파 |
| 승무원 사살 (관통 × 관통 누적) | 70% | 높음 | 차체·핵심 파츠 대부분 온전 |
| **승무원 전원 사망 + 차체 무력화** | **80~90%** | **최고** | 플레이어가 의도해야 도달하는 경로 |

### 5.2 "승무원 전원 사망" 개념

- 내부 관통 누적 → 내부 인원이 질식·연소·파편으로 **전원 전투 불능**
- 차체·파츠는 **물리적으로 온전**하지만 기동 불가
- 플레이어는 해당 적 기동을 본인이 원하는 시점에 중단시킨 뒤 **내부만 공략하는 플레이**를 설계할 수 있음
- 이것이 CRUX 빌드 최적화의 **최고 난이도 루트**

### 5.3 노획 산출 순서

```
전투 종료 →
  적 전차별 격파 방식 판정 (last impact + 누적 영향)
  격파 방식 계수 × 파츠 베이스 생존률 → 파츠별 생존 판정
  생존 파츠를 파츠 풀에 등록
  차체 노획 여부 판정 (Scout~Siege별 독립 확률)
  공용 인벤토리 투입 + 결과 요약 팝업
```

- 연쇄 손상 보정: 인접 슬롯이 폭발했으면 인접 파츠 생존 확률도 하락
- 차체 노획 → 부대 **신규 전차 슬롯** 개방 (부대 최대 규모 한도 내)

### 5.4 차체 노획과 부대 확장

- 부대 최대 전차 수: **1~6대** (스토리 진행에 따라 상한 확장)
- 차체 노획 시 즉시 부대 슬롯 점유. 상한 도달 시 기존 전차 폐기 선택 강제
- 폐기된 전차의 파츠는 공용 인벤토리로 환원 (사기 페널티 소폭)
- **로시난테는 폐기 불가**

## 6. 빌드 예시 — 초기 5인 부대

### 6.1 단독 출격 (Act 1 초반)

- **로시난테 (Assault)** — 초기 상태 (§9 참조)
- 5인 전원 탑승
- 파츠 교체 불가 (초기 고정)
- 임무: 생존, 첫 스테이지 클리어

### 6.2 2대 편성 (Act 1 중반)

- **로시난테 (Assault)** + **노획한 Scout 1대**
- Scout는 승무원 부족 상태로는 **출격 불가** (docs/04 §8.1, 2026-04-19 개정)
- Act 1 중반 스토리에서 신규 승무원 합류 이벤트 → 완전 편성되면 출격 해금. 그 전까지는 보관 상태 유지
- 전술 운용: 로시난테가 주력, Scout가 정찰·측면 기동

### 6.3 3~4대 (Act 2)

- Support 또는 Heavy 추가 노획
- 승무원 풀 확장 (수작업 20명 중 6~12명 가용)
- 빌드 다양화 시작 — 스테이지별 파티 교체 전략

### 6.4 최종 (Act 3)

- 5~6대 풀 편성
- Siege 1대 보유 가능 (스토리 보상 또는 극 확률 노획)
- 모든 차체 5종 운용 가능성

## 7. 중량·출력 상세

### 7.1 하중 한도 (차체별)

| 차체 | 하중 한도 | 기준 엔진 출력 |
|---|---|---|
| Scout | 60 | 80 |
| Assault | 100 | 100 |
| Support | 110 | 95 |
| Heavy | 160 | 120 |
| Siege | 220 | 150 |

*수치는 초안. 밸런스 패스에서 확정.*

### 7.2 과적 페널티

하중이 한도 초과 시:
- 이동 AP +1 (셀당)
- 엔진 과열 누적 속도 1.5배
- 캐터필러 내구 소모 2배

경계값 5% 이내 사용 권장 — 과적은 장기 플레이에서 자원 낭비.

### 7.3 출력 여유 활용

```
여유 출력 = 엔진 출력 - (차체 요구 + 파츠 출력 수요 합)
```

- 여유 ≥20 → 중장갑 추가 장착 가능 (리액티브 등)
- 여유 ≥30 → 부스터 스킬 활성화
- 여유 < 0 → 장착 불가

## 8. 데이터 스키마 개요

### 8.1 `TankDataSO` (Crux.Data)
```
id, displayName, hullClass : HullClass    // Scout/Assault/Support/Heavy/Siege
weightCapacity, powerRequirement
slotTable : SlotCount (카테고리별 수량)
isRocinante : bool                        // 로시난테 특례 플래그
```

### 8.2 `PartDataSO` 계열
```
id, category : PartCategory
weight, powerDraw
specificationTags : string[]              // 규격 호환성 체크용
grade : PartGrade                         // Trash/Common/Skilled/Elite/Legendary
```

카테고리별 세분화:
- `EngineDataSO` (power 생산, 과열)
- `TurretDataSO` (회전, 수용 구경)
- `MainGunDataSO` ← 기존 `TankDataSO` 내 주포 속성
- `AmmoDataSO` (기존)
- `ArmorDataSO` (face별)
- `TrackDataSO` (기동, 내구, 지형)
- `AuxiliaryDataSO` (보조 장비)

### 8.3 `TankInstance` (Crux.Unit, 런타임)
```
tankData          : TankDataSO
installedParts    : Dictionary<SlotKey, PartInstance>
currentWeight     : float
currentPowerDraw  : float
compatibilityFlags: { weightOk, powerOk, specOk }
```

### 8.4 `ConvoyInventory` (Crux.Unit 또는 Crux.Core)
```
partStash      : List<PartInstance>       // 공용 인벤토리
tankSlots      : TankInstance[]           // 부대 전차 1~6대
maxTankSlots   : int                      // 스토리 진행도 종속
```

## 9. 로시난테 — 플레이어 전차 초기 스펙

> 2026-04-15 추가. 승무원 시스템(docs/04)과 맞물려 전개되는 주인공 전차의 초기 상태와 성장 로드맵.

### 9.1 서사 정당성

로시난테는 180년 전 군수 창고에서 발견된 **고장·파손 상태의 재래식 중형전차**다. 프리테나·이리스·아스트라 일행이 **지상 탐색으로 주워 모은 각기 다른 출처의 부품들을 때려 넣어** 겨우 구동 가능한 상태로 만든 것이 첫 출격 상태다. 차체와 중량 등급만 중형일 뿐, 주포·엔진·장갑·탄약고는 모두 출처·규격이 제각각인 임시 조립품이다.

### 9.2 의도적 불일치 — 승무원 선호와 초기 장착

승무원의 선호 항목(docs/04 §2.2)은 로시난테 초기 장착과 **의도적으로 어긋난다**:

| 승무원 | 선호 | 초기 장착 | 결과 |
|---|---|---|---|
| 리리드 (포수) | 중구경 주포 | 임시 소구경 직사포 | 선호 보너스 미발동 |
| 프리테나 (조종수) | 중형 중량대 | (중형 중량 유지) | 선호 보너스 적용 ← 유일 |
| 그린 (탄약수) | AP탄 | AP탄 단종 휴대 | 선호 보너스 적용 |
| 이리스 (기총/수리) | 엔진 수리 | 결함투성이 엔진 | 수리 수요는 많지만 복구 기반 부족 |
| 아스트라 (전차장) | 중형전차 | (중형 차체) | 선호 보너스 적용 |

→ 플레이어가 파츠를 교체·복구해나가는 여정이 곧 **승무원들의 진정한 각성 과정**이 된다. 정비 진행 = 서사 성장 = 기계적 능력치 상승의 3중 동기화.

### 9.3 초기 스펙 (게임 시작 시점)

| 부품 | 초기 상태 | 게임 효과 |
|---|---|---|
| **차체** | Assault (중형전차) 고정 | 변경 불가. 로시난테의 정체성 |
| **엔진** | 낡은 저출력 | 최고 속도 70%, 과열 위험 상시 |
| **포탑** | 수동 회전 | 회전 AP +1 |
| **주포** | 임시 소구경 직사포 | 관통력·기본 명중 낮음, 대중형 표적 무력 |
| **탄약** | AP 소용량 | 장탄수 제한, 탄종 1종만 휴대 |
| **장갑** | 전면 부분 장갑 + 측후면 노출 | 측·후면 피격 시 관통 확률 급증 |
| **캐터필러** | 마모 상태 | 파손 확률 +10% |
| **기총** | 구형 경기총 1정 | 대 드론·보병 기본 대응 수준 |
| **보조 슬롯** | 비어 있음 (+1 특례 슬롯 포함) | 장비 획득 시 즉시 장착 가능 |

**요약**: 모든 축에서 기본값 이하. 성장 기대치가 가장 큰 케이스. 사용자가 개조하는 모든 선택이 곧바로 체감된다.

### 9.4 성장 로드맵 (3 막)

**Act 1 — 생존** (초반 스테이지)
- 복구: 엔진 1차 수리, 캐터필러 교체, 장갑 측면 보강
- 해금: 중구경 주포 입수 → **리리드 선호 발동** (명중 +10)
- 서사: 지하 마을을 떠나 첫 생존 전투

**Act 2 — 반격** (중반 스테이지)
- 복구: 후면 장갑 완성, 엔진 고출력 교체, 중·대구경 전환 가능
- 해금: 다연장·특수 탄종 → **그린 탄종 숙련 확장**, **탄약수 특화 스킬 다수 활성**
- 서사: 바이오 코어 거점 공격

**Final — 완성된 로시난테**
- 복구: 전 부품 최상위
- 해금: 전차장 고급 지휘 스킬 트리 완성, 5인 전원 마크 5 근접
- 서사: 최종 바이오 코어 돌입

### 9.5 구현 관점 — 후속 작업

- 초기 파츠의 구체 수치(관통력·장갑값·최고 속도 등)는 밸런스 패스에서 확정
- `TankDataSO`에 **"로시난테 초기" 프리셋 자산** 필요 (신규, isRocinante=true)
- 파츠 교체 UI는 **정비 씬** (Phase 2) 구축 시 함께 제작
- 교체된 파츠가 승무원 선호와 일치/불일치할 때의 UI 피드백 필요 (정비 씬에서 매칭 프리뷰)
