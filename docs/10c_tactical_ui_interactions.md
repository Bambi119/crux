# 10c. 전술 맵 UI 인터랙션 흐름

> 작성: 2026-04-23  
> 설계 참고: Front Mission 1 WEAPON SELECT 화면  
> 관련 문서: [`10_ui_ux.md`](10_ui_ux.md) (전체 구조) · [`06_combat_system.md`](06_combat_system.md) (전투 경제) · [`04b_skill_system.md`](04b_skill_system.md) (승무원 스킬)

## 0. 개요

전술 맵 위에서 아군 유닛 선택 → 커맨드 박스 표시 → 이동/사격/방향전환/스킬/대기 중 선택하는 일련의 UI 흐름을 정의한다.

**핵심 원칙**:
- 모든 팝업 및 선택 메뉴는 **화면 경계 플립**을 적용하여 오프스크린 방지
- 현재 상태·예상 결과를 실시간으로 표시 (명중률, 예상 데미지, AP 소모량)
- 뒤로 가기(취소) 경로가 각 단계에서 명확하게 제공될 것

---

## 1. 유닛 선택 단계

### 1.1 선택 방법

| 입력 | 결과 |
|---|---|
| **A/B/C/D 키** | 아군 유닛 배열 순서대로 개별 선택 (0~3 인덱스) |
| **마우스 좌클릭** | 마우스 위치 그리드 셀의 아군 유닛 선택 (명확하지 않으면 가장 가까운 유닛) |

### 1.2 선택 시각 피드백

- **선택 마커**: 유닛 머리 위에 화살표 또는 타일 색상 **밝게 강조** (옵션, Phase 3 프론트엔드에서 최종 결정)
- 이미 행동을 마친 유닛은 **회색 처리 또는 닫힌 상태 아이콘** 표시

### 1.3 선택 가능 조건

- 아직 턴 AP를 사용하지 않은 아군 유닛만 선택 가능
- 같은 턴 내 이미 선택했던 유닛 재선택 가능

---

## 2. 커맨드 박스 (명령 선택 메뉴)

### 2.1 표시 위치 및 경계 처리

선택한 유닛의 **인접 타일** 또는 **유닛 옆(월드 좌표 → 스크린 좌표 변환 후)**에 팝업.

**화면 경계 플립 규칙**:
- 유닛이 화면 **우측에 가까우면** → 박스를 **좌측**에 표시
- 유닛이 화면 **좌측에 가까우면** → 박스를 **우측**에 표시
- 유닛이 화면 **상단에 가까우면** → 박스를 **하단**에 표시
- 유닛이 화면 **하단에 가까우면** → 박스를 **상단**에 표시

여유 공간 판정은 **Box collider의 예상 크기(약 160×180px) + 안전 마진(20px)**를 고려.

### 2.2 메뉴 항목 (고정 순서)

```
1. 이동
2. 사격
3. 방향전환 (NEW)
4. 전차장 스킬
5. 대기
6. 취소
```

메뉴 내비게이션:
- **↑/↓ 키** 또는 **마우스 호버**: 항목 포커싱
- **Space 키** 또는 **마우스 좌클릭**: 선택 확정
- **Tab 키** 또는 **마우스 우클릭**: 취소 → 유닛 선택 상태로 복귀

---

## 3. 이동 흐름

### 3.1 이동 가능 영역 표시

"이동" 선택 후:
1. 현재 유닛의 **남은 AP로 도달 가능한 모든 셀**을 계산
2. 해당 셀들을 **파란색 타일 하이라이트** 표시
3. 현재 유닛 위치는 **더 밝은 색** 강조

### 3.2 타일 선택 및 AP 미리보기

- **WASD 키** 또는 **마우스 호버**: 이동 대상 셀 선택
- 선택된 셀 위에 **모달 팝업** 표시:
  ```
  소비 AP - 남은 AP / 총 AP
  예: 2 - 6 / 8
  ```
- 모달은 타일과 함께 이동하거나, 화면 일정 위치에 고정 (Phase 3에서 결정)

### 3.3 이동 취소 및 확정

| 입력 | 동작 |
|---|---|
| **Space 키** 또는 **좌클릭** | 선택 타일로 이동 확정 → 유닛 이동 실행 |
| **Tab 키** 또는 **우클릭** | 이동 취소 → 커맨드 박스로 복귀 |

### 3.4 이동 완료 후 자동 재표시

이동 실행 후 **자동으로 커맨드 박스 재표시** (같은 유닛).

**단, 사거리 내 적 유무에 따른 분기**:

| 상황 | 표시되는 메뉴 |
|---|---|
| 사거리 내 적 **있음** | 모든 항목 (이동/사격/방향전환/스킬/대기/취소) |
| 사거리 내 적 **없음** | 사격 제외 (이동/방향전환/스킬/대기/취소) |

사거리 계산:
- 주포 최대 사거리 + 기관총 최대 사거리의 합 범위 내 적이 있는지 체크
- 지형 엄폐·LOS 미고려 (전 방향 탐색)

## 4. 방향전환 (신규 기능)

### 4.1 개요

제자리 차체 회전. AP를 소모하며, 현재 선택 커맨드에서 접근 가능한 별도 미니 UI 제공.

### 4.2 AP 비용

**60° 당 1 AP** (Option A) — `GridTankUnit.RotateHullInPlace(float deltaDegrees)` 구현 완료.

공식: **AP 비용 = ⌈|회전각|/60⌉**

예시:
- 0° → 60°: 1 AP
- 0° → 120°: 2 AP
- 0° → 30°: 1 AP (0.5 올림)

기존 `GridTankUnit.RotateToDirection()` 회전 규칙과 정합.

### 4.3 UI 흐름

"방향전환" 선택 후:
1. 유닛 주변에 **6방향 방사형 UI** 표시 (6각형 중심이 유닛)
2. **WASD 키** 또는 **화살표 키**: 회전 방향 지시
3. 선택된 방향 위에 **"회전 → 1 AP"** 같은 비용 표시 (현재 방향과의 차이 기반)
4. **Space 키** 또는 **확정 버튼**: 회전 실행
5. **Tab 키** 또는 **취소 버튼**: 회전 취소 → 커맨드 박스로 복귀

회전 완료 후: **커맨드 박스 재표시** (이동과 동일 규칙, 사거리 재계산)

---

## 5. 사격 흐름

### 5.1 타겟팅 시작 및 자동 선택

"사격" 선택 → 순차 자동 타게팅:
1. 현재 유닛이 사격 가능한 **모든 적 유닛 중**가장 가까운 적을 자동 선택
2. 카메라가 **해당 적 유닛으로 부드럽게 패닝** (0.3초 정도)
3. 우측 `UnitInfoCard` 갱신 → 적 정보 표시 (HP, 장갑, Trait 등)

### 5.2 타겟 순환

| 입력 | 동작 |
|---|---|
| **W/A/S/D 키** | 타겟 순환: 다음/이전 적 선택 |
| **마우스 휠 상/하** | 타겟 순환 (키보드 대체) |
| **마우스 좌클릭 (적 타일)** | 직접 클릭으로 타겟 변경 |

타겟 변경할 때마다:
- 카메라 **패닝 애니메이션** (새 타겟으로 부드럽게 이동, 0.2~0.3초)
- `UnitInfoCard` 즉시 갱신

### 5.3 타겟 확정

| 입력 | 동작 |
|---|---|
| **Space 키** | 타겟 확정 → 무기 선택 단계로 진행 |
| **좌클릭 (타겟 적)** | 타겟 확정 (마우스로도 가능) |
| **Tab 키** 또는 **우클릭** | 사격 취소 → 커맨드 박스로 복귀 |

---

## 6. 무기 선택 (사격 후단계)

### 6.1 무기 선택 박스 출현

타겟 확정 후:
1. **무기 선택 커맨드 박스** 출현 (타겟 유닛 옆, 경계 플립 적용)
2. 아군 유닛이 장착한 무기 목록 표시:
   - 주포 (탄종별 슬롯, 여러 개 가능)
   - 기관총 (MG)
   - 연막탄 (Smoke)
   - 취소

### 6.2 사거리 제약 시각 표현

- **사거리 내 무기**: 정상 색상·선택 가능
- **사거리 외 무기**: **회색 처리** (선택 불가, 호버 시 "사거리 외" 툴팁)

사거리 판정: 아군 유닛 ↔ 타겟 유닛 간 육각 거리 계산

### 6.3 실시간 프리뷰 (UnitInfoCard)

무기 선택 직후 및 선택 중:
- **명중률**: 현재 선택 무기의 명중 확률 수치 (% 단위)
- **예상 데미지**: 평균 데미지 수치
- **부위 바 깜빡임**: 예상 피격 부위별로 **`damageRatio = expectedDamage / partMaxHP`** 만큼의 구간만 하이라이트 깜빡임
  - 예: Turret 최대 100 HP, 예상 데미지 30 → 부위 바의 70~100 구간(폭 30%)만 깜빡임
  - **전체 부위 바 점멸 금지** — 피격 예상 부위의 해당 구간만 시각화

### 6.4 무기 선택 및 취소

| 입력 | 동작 |
|---|---|
| **↑/↓ 키** 또는 **호버** | 무기 메뉴 내 포커싱 |
| **Space 키** 또는 **좌클릭** | 무기 선택 확정 → 배틀 씬(FireActionScene) 전환 |
| **Tab 키** 또는 **우클릭** 또는 **ESC** | 무기 선택 취소 → 타겟팅 단계로 복귀 (타겟 유지) |

## 7. 입력 요약표

전체 흐름의 키보드/마우스 매핑:

| 단계 | 입력 | 동작 |
|---|---|---|
| **유닛 선택** | A/B/C/D | 아군 유닛 개별 선택 |
| | 좌클릭 (아군 타일) | 마우스 위치 셀 선택 |
| **커맨드 박스** | ↑/↓ | 메뉴 항목 포커싱 |
| | Space / 좌클릭 | 메뉴 항목 선택 |
| | Tab / 우클릭 | 취소 → 유닛 선택 |
| **이동 — 타일 선택** | WASD | 이동 대상 타일 선택 |
| | 호버 (마우스) | 이동 대상 타일 프리뷰 |
| **이동 — 확정** | Space / 좌클릭 | 이동 확정 |
| | Tab / 우클릭 | 취소 → 커맨드 박스 |
| **방향전환** | WASD / 화살표 | 회전 방향 지시 |
| | Space | 회전 확정 |
| | Tab / 우클릭 | 취소 → 커맨드 박스 |
| **사격 — 타겟팅** | WASD / 휠 | 타겟 순환 |
| | 좌클릭 (적 타일) | 직접 타겟 선택 |
| | Space / 좌클릭 | 타겟 확정 → 무기 선택 |
| | Tab / 우클릭 | 취소 → 커맨드 박스 |
| **무기 선택** | ↑/↓ | 무기 목록 포커싱 |
| | Space / 좌클릭 | 무기 선택 확정 → FireActionScene |
| | Tab / 우클릭 / ESC | 취소 → 타겟팅 |

## 8. 특수 경우 및 제약

### 8.1 AP 부족

- 이동: 남은 AP 부족 시 해당 타일 **선택 불가**(회색 처리)
- 사격: 필요 AP 부족 시 무기 **회색 처리**
- 방향전환: 필요 AP 부족 시 해당 방향 **회색 처리**

### 8.2 사거리 제약

- 사격: 모든 무기가 사거리 외면 → 사격 메뉴 자체 **회색 처리**
- 무기 선택: 특정 무기만 사거리 외면 → 해당 무기 **회색 처리**

### 8.3 스킬 가용성

전차장 스킬 (메뉴 항목 4번) 선택 시:
- 현재 스킬 쿨다운·AP 확인 후 선택 가능 여부 판정
- 스킬 상세 규칙은 `04b_skill_system.md` 참조

### 8.4 대기 (턴 종료)

"대기" 선택:
- 현재 유닛의 턴 종료
- 남은 AP는 **다음 턴으로 미적용** (턴 경계마다 리셋)
- 자동으로 **다음 선택 가능 아군 유닛**의 커맨드 박스 표시

---

## 9. 설계 원칙 요약

1. **경계 플립**: 모든 팝업이 화면 밖으로 나가지 않도록 자동 위치 조정
2. **실시간 피드백**: 명중률·데미지·AP 소모를 선택 직후 표시
3. **뒤로 가기 명확성**: 각 단계마다 취소 경로(Tab/우클릭)가 존재
4. **계층적 진행**: 유닛 선택 → 커맨드 → 세부 행동(이동/사격 등)의 명확한 위계
5. **마우스·키보드 동시 지원**: 모든 선택이 두 입력 방식으로 가능
6. **부위별 데미지 시각화**: 전체 바 깜빡임 대신 **예상 피격 구간만 하이라이트**
