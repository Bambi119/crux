# 4. 승무원 시스템

> 개정: 2026-04-15 — 턴제 Hex 전술 전투 기반 완전 재작성
> 이전 판본은 실시간 서바이벌 가정이었으나 실제 구현은 턴제이므로 AP·마크·사기 중심으로 재정의.

## 문맥 — 부대 단위 운용

CRUX는 **부대 단위**(Convoy) 게임이다. 플레이어는 1~6대의 전차를 운용하며 각 전차는 5명의 승무원을 갖는다. 승무원 풀(MVP 20명 수작업)은 부대 전체 공용이며, 정비 씬에서 자유롭게 재배치 가능.

- **최대 부대 규모**: 스토리 진행도에 종속 (초반 1~2대 → 후반 4~6대). 차체 노획과 스토리 해금에 연동
- **승무원 이동**: 정비 씬에서 전차 간 자유 이동. 출격 중에는 고정
- **공석**: 슬롯이 하나라도 비면 **출격 불가** (§8.1, 2026-04-19 개정)
- **이번 출격 파티**: 스테이지별로 부대 중 1~N대 선택 출격 (docs/03 §1.3)

승무원 시스템이 작동하는 무대는 단일 전차가 아니라 **여러 전차가 공존하는 부대 경제**라는 점에 유의. 파츠·승무원 배치는 부대 전체의 전술 유연성에 기여한다.

## 설계 원칙

1. **직책 = 불변 클래스** — 전차장은 포수가 될 수 없다. 각 직책은 고유 책임 축을 가진다.
2. **마크 숙련도** — 숫자 매트릭스 대신 0~5 마크 개수로 표현. 해상도 낮되 기억에 남는다.
3. **캐릭터는 고유 인물** — 수작업 20명 풀 기반. 기본 스탯·특성은 고정, 성장은 마크와 스킬 획득으로.
4. **스킬 블라인드 획득** — 플레이어는 타이밍은 알되 내용은 모른다. 풀이 좁아도 매 플레이 다른 조합 경험.
5. **AP·턴 단위 상수** — "-10%" 같은 연속 시간값 금지. 모든 효과는 턴제에 종속된 관측 가능한 상수.

## 1. 직책(Class) 5종

5인 승무원 구성. 각 직책은 전차 1대에 1명씩 배치. 직책은 캐릭터 생성 시 확정되며 변경 불가.

| 직책 | 기계적 책임 | 주 숙련도 축 |
|---|---|---|
| **전차장(Commander)** | 사기 관리 · 지휘 버프 · 고급 대응·정보 | 전차 클래스별 (4축: 경·중·중·초중) |
| **포수(Gunner)** | 주포 명중률 · 기본 오버워치 | 주포 구경별 (3축: 소·중·대) |
| **탄약수(Loader)** | 장전 AP 절약 · 탄종 보정 | 탄종별 (3축: AP·HE·로켓) |
| **조종수(Driver)** | 이동 AP 절약 · 후진·회전 | 차체 중량대별 (3축: 경·중·초중) |
| **기총사수/수리병(Gunner-Mech)** | 기총 · 모듈 수리 · 대 소프트타겟 | 수리 모듈별 (3축: 캐터·엔진·포탑) |

### 1.1 전차장
- 3 축의 관리자: **사기** / **지휘** / **대응·정보**
- 전차 전체에 파생 효과 (사기 게이지·지휘 버프·고급 반응)
- 기본 오버워치는 포수 영역, 전차장은 고급 대응(매복·약점 지적·경고 사격)만 담당

### 1.2 포수
- 주포 명중률 기반값 결정 + 기본 오버워치 패시브 보유
- 주포 구경별로 숙련도가 분리됨 — 대구경 전문 포수는 소구경 기관포에서 명중 보너스 없음

### 1.3 탄약수
- 장전 AP 절약·탄종 교환 시간 단축이 핵심 책임
- 대구경·다연장 전차에서 효과가 극대화, 소구경 자동급탄 주포에서는 의미 약함
- 탄종 특화로 HE 폭발 범위 / AP 관통력 / 로켓 유도 성능에 보정

### 1.4 조종수
- 이동 AP 절약, 후진 페널티 제거, 회전 턴 단축
- 차체 중량대별 숙련도 — 경전차 고속 기동 빌드의 코어

### 1.5 기총사수/수리병
- 기총 운용 + 전투 중 모듈 수리 겸임
- 대 드론·대 보병 소프트타겟 전문
- 버스트 연장·전탄 발사·제압 사격 같은 기총 특화 스킬 풀

## 2. 스탯 시스템

### 2.1 기본 스탯 3 (1~100, 캐릭터 고정값)

| 스탯 | 의미 | 적용 방식 |
|---|---|---|
| **명중 (Aim)** | 사격 정확도 기반값 | `baseHit + Aim × 0.5%` (포수·기총사수 주) |
| **반응 (React)** | 오버워치·반응 사격 트리거·명중 | 발동형 스킬 성능 |
| **기술 (Tech)** | 스킬 장착 가능량·수리·재장전 | `슬롯 가감`·수리 시간 |

**직책별 시작값 권장치**:
- 포수: 명중 60+, 반응 50+
- 탄약수: 기술 60+, 명중 30~40
- 조종수: 반응 55+, 기술 40~50
- 기총/수리: 명중 50, 기술 55+
- 전차장: 반응 55+, 명중 40+ (직접 사격보다는 관리자)

스탯은 **캐릭터 고정**. 성장으로 오르지 않음. 성장은 마크와 스킬 획득이 담당.

### 2.2 선호 항목 (직책별 1~2개)

| 직책 | 선호 | 보너스 |
|---|---|---|
| 전차장 | 전차 클래스 | 해당 클래스 탑승 시 시야 +1 |
| 포수 | 주포 종류 | 해당 무기 명중 +10, 듀얼 AP -1 |
| 탄약수 | 탄종 | 해당 탄 장전 AP -1 |
| 조종수 | 중량대 | 해당 중량대 이동 AP -1 |
| 기총/수리 | 수리 모듈 | 해당 모듈 수리 턴 절반 |

### 2.3 특성(Trait)

**캐릭터당 장점 1 + 약점 1 고정**. 생성 시 결정, 제거 불가 (MVP).

**장점 예**:
- **냉정함** — HP ≤30%에서도 명중 감소 없음
- **카리스마** (전차장 전용) — 모든 사기 하락 이벤트 효과 절반
- **사격광** — 주포 사격 성공 시 반응 +10 (다음 턴)
- **정비의 신** (기총/수리 전용) — 수리 시간 추가 절반
- **기관의 천재** (탄약수 전용) — 장전 AP 절약 +1

**약점 예**:
- **유폭 공포** — 탄약고 유폭 목격 후 1턴 명중 -20
- **편식** — 선호 무기·탄종 외 사용 시 명중 -5
- **고집** — 직접 명령만 수행, 스킬 쿨다운 +1
- **비관** — 사기 하락 이벤트 효과 1.5배

## 3. 마크 숙련도 시스템

### 3.1 기본 구조

- **공통 axis (성장 축)** — 탑승 전차의 적 격파 1건당 차내 승무원 전원 +1 카운트
- **직책별 axis 분리 폐지** (2026-04-22 결정) — 활용 기회 격차로 보직 간 성장 페이스 불균형이 생기는 문제 해소. 메타게이밍(예: 조종수가 회피 카운트 누적을 위해 적성 무관 경전차 강제 탑승) 압력도 제거
- **직책별 마크 효과 분기는 유지** — axis는 공통이지만 마크업 결과 보너스는 직책마다 다르게 적용 (§3.2)
- **직책별 카드 풀 분기는 유지** — 마크업 시 추출되는 스킬 카드는 직책 풀에서만 (`docs/04b §1.2 직책 종속 원칙`)
- 마크 상한 **5**
- 직책별 심볼: 전차장 ♛ · 포수 ⊕ · 탄약수 ◆ · 조종수 ▶ · 기총/수리 ✚
- 캐릭터 시트 한 줄 예: `제시카 · 포수 · ⊕⊕⊕⊕ (대구경 명중 +14)`

> **차 간 양보 메타 = 의도된 설계** — 강한 적은 숙련 부대가, 약한 적은 신규 충원 부대가 정리. 차 간 표적 선택의 전략 자유도로 활용

### 3.2 마크별 수치 효과

| 마크 | 주 스탯 보너스 |
|---|---|
| 1 | +3 |
| 2 | +6 |
| 3 | +10 |
| 4 | +14 |
| 5 | +20 |

**직책별 적용 대상**:
- 전차장 → 사기 기본값 · 사기 자연 회복량
- 포수 → 해당 구경 주포의 명중률
- 탄약수 → 해당 탄종 장전 AP 절감 · 탄종 효과 보정
- 조종수 → 해당 중량대 이동 AP 절감 · 회전 효과
- 기총·수리 → 해당 모듈 수리 속도 · 기총 명중률

### 3.3 획득 조건 (이중 게이트)

어뷰징 방어를 위해 격파 카운트와 전투 참전 수를 모두 요구.

```
마크 N 달성 조건:
  공통 axis 누적 격파 ≥ killThreshold[N]
  AND
  탑승 전투 수 ≥ battleThreshold[N]

killThreshold    = [1, 5, 15, 35, 70]    ⚠ 잠정
battleThreshold  = [1, 2, 5, 10, 20]     ⚠ 잠정
```

**카운트 규칙 (단순화)**:
- 격파를 낸 차의 승무원 전원 +1
- 차 단위 단일 카운트 (차내 승무원 간 기여도 분리 없음 — 1대가 1대 격파 = 5명 모두 +1)
- 다른 차의 승무원은 0 (차 간 양보 메타는 §3.1 의도된 설계)

> **⚠️ 임계값 잠정 — 재튜닝 대상**
> 위 두 임계값은 **직책별 axis 분리 시절(보직마다 카운트 분산)** 에 산정된 값. 공통 axis 채택 후 전원이 동시에 격파 카운트를 받으므로 이 페이스는 **마크 5 도달 시점이 너무 빠를 가능성**. 정확한 재튜닝은 `docs/09 맵 & 스테이지` Act별 적 등장 수가 확정된 이후 (`docs/07 §11 오픈 이슈` 추가 항목).

### 3.4 가시성 원칙

- **공개**: 다음 마크까지 남은 진행도 (예: `⊕ 대구경 14/15`)
- **블라인드**: 다음 획득 스킬의 이름·효과

즉 "언제" 각성하는지는 알되 "무엇이" 나올지는 모른다.

### 3.5 고유 트레잇 (Unique Trait, 고유 특성)

> 마크와 **독립된 별개 트랙**. 캐릭터별 1~2개. 공통 axis(격파)로는 손에 들어오지 않는 캐릭터 색채를 부여하기 위한 장치.

#### 3.5.1 정의

- **트레잇** = 캐릭터 백스토리에서 파생되는 **고유 패시브/조건부 효과**
- **고유 스킬** = 트레잇이 부여하는 캐릭터 전용 스킬 1개. 직책 풀과 독립
- **활성 조건** = 캐릭터별로 다른 **개별 axis (트레잇 axis, 고유 성장 축)**. 마크 axis(격파)와 별도 카운트

#### 3.5.2 진행 흐름

```
캐릭터 영입 시
  └ 트레잇 1~2개 부착 (캐릭터 정의에 종속, 플레이어 선택 불가)
  └ 고유 스킬 잠금 상태로 시작

전투 중
  └ 트레잇 axis 조건 행동 → 트레잇 카운트 +N
       (마크 axis와 동시 누적, 서로 영향 없음)

조건 임계값 도달
  └ 정비 화면 진입 시 알림 (각성 카드와 별도 슬롯)
  └ 고유 스킬 활성화 (자동 장착, 슬롯 외 별도 소켓)
```

#### 3.5.3 슬롯 처리

- 고유 스킬은 **직책 슬롯 4개와 별개의 "트레잇 소켓" 1~2개** 에 장착 (직책 풀 슬롯을 점유하지 않음)
- 따라서 트레잇 활성으로 인해 직책 스킬을 포기할 필요 없음

#### 3.5.4 그라인드 (반복 플레이) 요소

- 트레잇 axis는 **공통 axis보다 빈도가 낮은 행동**으로 설계 (예: 유폭 10회 / 시야 외 적 명중 5회 / 방어 자세에서 격파 8회)
- 캠페인 1회 클리어로 모든 캐릭터의 트레잇이 풀리지 않음 → **반복 플레이 동기**
- 운영 효과: 컨텐츠 수명 연장, 캐릭터별 빌드 다양화

#### 3.5.5 예시 (잠정)

|---|---|---|---|
| 티티 | 방화광 | 탑승 전차의 명중 판정 시 화염 확률 +5% | 탑승 전차 적 유폭 10회 |

#### 3.5.6 MVP 범위 (잠정)

- 캐릭터 풀 **10~15명**, 인당 트레잇 **1개** 우선
- 트레잇 2개 캐릭터(주인공급 등)는 Phase 2 또는 백스토리 분량 검토 후
- 본 작업은 어시스턴트가 캐릭터별 후보안을 제출하고 사용자가 채택 (`docs/02` 진행과 병행)

## 4. 스킬 시스템

> **권위 문서**: [`04b_skill_system.md`](04b_skill_system.md) — 스킬 타입·요구조건·슬롯·풀·획득 파이프라인 통합.

핵심 요지 (상세는 04b 참조):

- **타입 3계층** — Passive (장착 시 항상, 사격 토글 변형 포함) · Triggered (발동형 — 조건+확률 자동) · Active (명령 커맨더 — 전차장·수리병만). → 04b §2
- **요구조건 Predicate** — MainGunCaliber / MainGunMechanism / AmmoType / MGType / HullClass × Any / All / None. → 04b §3.2
- **슬롯** — 직책당 4슬롯 고정. 포수·탄약수·조종수: P2+T2 / 전차장·수리병: P2+T1+A1. → 04b §3.1
- **범용 vs 특화** — 범용은 요구조건 없음·효과 보통, 특화는 요구조건·효과 강력. 전차 교체로 요구조건 상실 시 **비활성화** + UI 경고. → 04b §3.3
- **마크 스케일링 분기** — 발동형은 마크업 시 강화 (발동률·범위 상승), 패시브·액티브는 마크업 시 신규 카드 획득. → 04b §4
- **획득 파이프라인** — 마크 레벨업 시 풀 쿼리 → 후보 수 분기 (0 스킵 · 1 자동 · 2~3 카드 · 4+ 추출). 정비 씬 처리. → 04b §7

## 6. 전차장 특화 시스템

### 6.1 사기(Morale) 시스템

**척도 0~100. 전차 전체에 적용되는 단일 값.**

| 구간 | 상태 | 효과 |
|---|---|---|
| 80~100 | 사기충천 | 전 승무원 명중 +5, 전투 시작 시 스킬 쿨다운 -1 (1회) |
| 50~79 | 정상 | — |
| 25~49 | 흔들림 | 전 승무원 명중 -5, 반응 사격 발동 실패 20% |
| 0~24 | 공황 | 전 승무원 명중 -15, 액티브 스킬 금지, 매 턴 AP -1 |

**전투 시작 사기**:
```
base 50 + (전차장 해당 클래스 마크 × 3)
```
→ 마크 5 전차장: 시작 사기 65 (정상 상위권)

**매 턴 시작 자연 회복**: `전차장 마크 × 1`

**사기 변동 이벤트**:
| + | − |
|---|---|
| 적 격파 +5 | 승무원 부상 -15 |
| 도탄(튕김) +3 | 모듈 파손 -5 |
| 관통 명중 +3 | 측면/후면 피격 -10 |
| 전차장 격려 사용 +10~20 | 탄약고 근접 피격 -20 |

**공황 안전장치**: 전투당 1회, 사기가 0~24 구간 진입 직후 자동 스크립트 이벤트 "전차장의 외침" 발동 → 사기 +15 회복. 데스 스파이럴 방지.

### 6.2 전차장 스킬 카테고리

전차장 스킬 풀은 4 카테고리 — **(A) 지휘(사기)** · **(B) 대응** · **(C) 정보** · **(D) 반격 보강**. 액티브는 명령 커맨더 슬롯에 장착, 발동형은 자동 발동.

→ 상세 표·요구조건·액티브 vs 발동형 분류: [`04b §5.5`](04b_skill_system.md#55-전차장-commander)

### 6.3 지휘 스킬 발동자·수혜자 구조

지휘 스킬은 **발동자 = 전차장**, **수혜자 = 타 승무원**으로 분리. 전차장 AP·쿨다운만 소모, 수혜자는 자동으로 혜택을 받는다 — UI는 수혜자 측 버프 아이콘으로 표시.

→ 발동/수혜 데이터 모델·이벤트 흐름: [`04b §5.6`](04b_skill_system.md#56-발동자수혜자-구조-지휘-스킬)

## 7. AP 경제 — 하이브리드 C

- **전차 공용 AP 8** — 이동·기본 사격·재장전은 여기서 소모
- **승무원별 스킬 쿨다운** — 스킬은 대부분 쿨다운(1~5턴) 자원으로 제약
- **일부 강력 스킬만 추가 AP 요구** — 듀얼(+5 AP), 전탄 발사(전량) 등

포수가 스킬 썼다고 탄약수 스킬 못 쓰는 경쟁 없음. 5명 모두 같은 턴에 스킬 사용 가능. 단 기본 행동(이동·사격)은 여전히 공용 AP 경쟁.

## 8. 부상 시스템

| 단계 | 발생 조건 | 페널티 | 회복 |
|---|---|---|---|
| 경상 | 피격 관통 시 10% 확률 | 다음 전투 스탯 -10% | 전투 1회 자동 |
| 중상 | 피격 관통 시 1% 확률, 탄약고 유폭 시 100% | 2~3 전투 참전 불가 | 정비 씬 대기 |
| 치명상 | 스토리 이벤트만 | 영구 사망 | — |

### 8.1 공석 = 출격 불가 (Hard Gate)

**기본 규칙 (2026-04-19 개정)**:
전차의 승무원 슬롯 수 = 해당 전차의 최소 요구 승무원 수.
하나라도 공석이면 **출격 자체가 불가**. Hangar에서 ▶출격 버튼이 거부된다.

- 경차(Scout): Commander / Gunner / Driver 3 슬롯 모두 채워야 함
- 중형(Assault): Commander / Gunner / Loader / Driver / GunnerMech 5 슬롯 전부
- 공석 상태 저장 가능(편성 작업 중), 단 전투 진입 차단

**이전 설계 대비**:
기존 "공석 페널티"(명중 -30, AP +1 등) 접근 폐기. 공석은 허용되지 않는 상태로 강화 — 승무원 확보는 전략적 병목으로 작동하며, 플레이어는 공석 상태로 운용하는 edge 케이스를 시도할 수 없다.

**중상자 취급**: 부상 중상은 공석과 동일 — 해당 전투 출격 불가 (docs §8.2).

### 8.2 치명상 정책

**치명상은 스토리 이벤트 한정**. 전투 결과(HP 0·유폭)로는 발생하지 않음. 플레이어가 관리·로딩으로 회피할 수 없도록 서사 구조에 묶인다.

## 10. 데이터 스키마 개요

### 10.1 `CrewMemberSO` (Crux.Data)
```
id                  string
displayName         string
portrait            Sprite
klass               CrewClass (enum)
aim, react, tech    int (1~100)
preferredTag        string         // 선호 항목 ID
traitPositive       TraitSO
traitNegative       TraitSO
startingMarks       Dictionary<AxisKey, int>
storyRef            string
```

### 10.2 `CrewSkillSO` (Crux.Data)
```
id, displayName, description
type                SkillType        // Passive / ActiveInstant / ActiveReactive
targetClass         CrewClass
requires            SkillRequirement[]
apCost              int
cooldown            int
supportsDual        bool
dualExtraCost       int
triggerType         ReactiveTrigger  // Move / Fire / Sight (Reactive 전용)
effectKey           string           // effect 실행 훅
```

### 10.3 `SkillRequirement`
```
axis                RequirementAxis  // MainGunCaliber / MainGunMechanism / AmmoType / MGType / HullClass
operator            Op               // Any / All / None
values              string[]
```

### 10.4 `TankCrew` (Crux.Unit, MonoBehaviour)
전차에 붙는 5슬롯 컨테이너.
```
commander, gunner, loader, driver, mgMechanic : CrewMemberRuntime
morale              int (0~100)
moraleState         MoraleBand
panicSafetyUsed     bool             // 전투당 1회 공황 회복
```

### 10.5 `CrewMemberRuntime`
런타임 상태. SO + 가변 부분.
```
data                CrewMemberSO
currentMarks        Dictionary<AxisKey, int>
killCounters        Dictionary<AxisKey, float>
battleCounters      Dictionary<AxisKey, int>
ownedSkills         CrewSkillSO[]
equippedPassives    CrewSkillSO[2]
equippedActives     CrewSkillSO[2]
injuryState         InjuryLevel
cooldowns           Dictionary<SkillId, int>
```

### 10.6 `MoraleSystem` (Crux.Unit)
```
ApplyEvent(MoraleEvent kind, int amount)
TickTurn()                         // 매 턴 시작 자연 회복
GetBand() → MoraleBand
CheckPanicSafety()                 // 공황 진입 시 1회 자동 +15
```
