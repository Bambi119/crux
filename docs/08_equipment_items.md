## 0. 용어 정의

| 용어 | 영문 | 정의 |
|---|---|---|
| 파츠 | Part | 전차에 **영구 장착**되는 모듈 (`docs/05 §8.2 PartDataSO`). 본 문서 범위 외 |
| 소모품 | Consumable | 1회 사용으로 소진되는 캠프·전투 외부 사용 아이템. 부대 인벤토리에 비축 |
| 장비 | Equipment | 본 문서에서는 "파츠 + 소모품"의 상위 포괄어로 사용. 별도 카테고리 아님 |
| 아이템 슬롯 | Item Stack | 같은 `consumableDataId`를 공유하는 비축 1줄. `count: int` 필드로 수량 표현 |

## 1. 설계 원칙

1. **"장착 vs 비축" 이분법** — 장착 가능한 것은 모두 파츠(`docs/05`). 비축은 소모품(본 문서). 중간 형태(영구 장비) 없음
2. **소모품은 캠프 외부 사용만** (MVP) — 전투 중 사용 UI 추가는 Phase 2. 전투 외부에 한정해 인벤토리·UX 단순화
3. **무게·전력 비용 없음** — 소모품은 부대 인벤토리에 무한 스택. `docs/05 §2 호환성 3중 체계` 적용 안 함
4. **수동 사용** — 자동 발동 소모품은 도입하지 않는다(예: "전투 시작 시 자동으로 응급 키트 소모"). 플레이어가 명시적으로 사용
5. **효과는 "기존 시스템 단축"** — 새 효과 카테고리를 만들지 않는다. 부상 회복·사기 회복·내구도 회복 등 **이미 존재하는 메커니즘을 가속**

## 3. 카테고리 분류

### 3.1 11 §3.3.1 기본 시장 SKU 5종 분류

| SKU | 분류 | 위치 | 비고 |
|---|---|---|---|
| 표준 캐터필러 (300) | **파츠** | `PartCategory.Track`, 일반 등급 | `docs/05 §8.2` |
| 응급 키트 (150) | **소모품** | 본 문서 §4.1 | 신규 카테고리 |
| 연막탄 발사기 (400) | **파츠** | `PartCategory.Auxiliary`, 일반 등급 | 발사 기능은 전투 중 액티브 (`docs/06` 후속) |
| 조준 보조 (600) | **파츠** | `PartCategory.Auxiliary`, 숙련 등급 | 명중률 패시브 |
| 적외선 스코프 (800) | **파츠** | `PartCategory.Auxiliary`, 숙련 등급 | 시야·연막 관통 패시브 |

**결론**: 5종 중 **1종(응급 키트)만 소모품**. 나머지 4종은 파츠 카테고리 확장으로 처리. PartCategory enum 신규 값 추가 없음(전부 `Track`/`Auxiliary` 재활용).

### 3.2 신규 카테고리 정의

소모품은 단일 enum `ConsumableType`로 통합:

```
ConsumableType {
  MedKit          # 응급 키트 (§4.1)
  // 후속 후보:
  // RationFeast   # 비상 회식 (사기 가속)
  // RepairKit     # 즉시 모듈 수리 (캠프 정비 단축)
  // FuelCache     # 비상 연료 (스토리 임의 시점, MVP는 도입 안 함)
}
```

MVP는 **`MedKit` 1종만** 도입. 후속 SKU는 §10 튜닝 훅으로 추적.

## 4. 소모품 SKU

### 4.1 응급 키트 (MedKit)

| 항목 | 값 |
|---|---|
| 표시명 | 응급 키트 (Med Kit) |
| 가격 (기본 시장) | 150 자금 (`docs/11 §3.3.1`) |
| 효과 | 대상 승무원 1명의 부상 단계 **1단계 즉시 회복** (Severe → Minor / Minor → None) |
| 사용 시점 | 정비 화면 (`docs/03b §6`) — 부상 승무원 카드에서 "응급 키트 사용" 버튼 |
| 사용 비용 | 자금 0 (구매 시 지불 완료) |
| 부작용 | 없음 |
| 중첩 | 같은 승무원에 연속 사용 가능 (Severe → Minor → None은 키트 2개) |
| 회복 잔여 시간 무시 | `injuryBattlesRemaining`(`docs/03c §2.5`) 즉시 0으로 |

**의의**: 03b §3.3 부상 회복은 무료지만 **시간(전투 횟수)**가 비용이다. 응급 키트는 그 시간을 자금으로 단축. 11 v1.1 "수리 유료, 부상 무료" 원칙과 어긋나지 않음 — 부상 자연 회복은 여전히 무료, 키트는 **자금 ↔ 시간** 교환.

### 4.2 후보 SKU (Phase 2)

도입 시점은 첫 빌드 외부 피드백 후:

| 후보 | 효과 안 | 가격대 |
|---|---|---|
| RationFeast (비상 회식) | 부대 전체 사기 +5 (회식 1단계 효과 즉시·캠프 의존 없음) | 250 |
| RepairKit (즉시 수리 키트) | 모듈 1개 Destroyed → Damaged 즉시 (1캠프 단축) | 200 |
| MoraleBoost (사기 음악) | 부대 전체 사기 +3 (광역 약효) | 180 |

본 표는 §10 튜닝 훅 6종과 함께 추적.

## 5. 사용 시점·UX

### 5.1 정비 화면 통합 (03b §6 후속 보강 권장)

`docs/03b §6 UI 레이아웃` 중앙 3섹션의 **부상 회복 섹션**에 소모품 사용 버튼 추가:

```
[부상 승무원 카드]
  이리스 — 중상 (잔여 2전투)
  ─────────────────────────
  [응급 키트 사용 (보유 3)]   ← 본 문서 §4.1
```

- 보유 0이면 버튼 회색 + "응급 키트 없음 — 시장에서 구매"
- 클릭 → 즉시 적용, 카드 갱신, 보유량 -1
- 모달 확인 없음 (실수 회복은 가능, 오용 위험 낮음)

### 5.2 사용 불가 시점

- **전투 중** — MVP 범위 외. 전투는 출격 전 부상 0 상태 가정
- **월드맵 노드 이동 중** — 이동 도중 사용 UI 없음. 캠프 도착 후 정비 탭에서만
- **타이틀** — 의미 없음

### 5.3 구매 UX (`docs/11 §3.3.1` 보강 권장)

기본 시장 화면에서 응급 키트는 다른 파츠와 **분리된 섹션**:

```
[ 시장 ]
─────────────────────────
파츠
  표준 캐터필러   300
  연막탄 발사기   400
  조준 보조       600
  적외선 스코프   800
─────────────────────────
소모품
  응급 키트       150  [구매 1]  [구매 5]
```

- 파츠는 1개 단위, 소모품은 1·5 묶음 구매 UX
- 5개 묶음은 단가 동일(할인 없음, MVP 단순화)

## 6. 인벤토리 모델

### 6.1 스택 표현

```
ConsumableStack {
  type: ConsumableType
  count: int                  # 0 이상, 0이면 스택 제거
}
```

- 같은 `type` 다중 스택 금지(중복 시 자동 병합)
- `count`만 변동, 인벤토리 슬롯 수 = `ConsumableType` enum 값 수

### 6.2 상한·정렬

- **상한 없음** (MVP). 무게·슬롯 제약 없음
- 정비 탭 표시 순서: enum 선언 순서 (현재 `MedKit` 단일이라 의미 없음)

### 6.3 손실·영구 효과

- 캠페인 전체에서 누적 비축 (전투 패배해도 소실 안 됨, MVP)
- Phase 2: "보급 라인 끊김" 같은 스토리 이벤트 시 일부 소모 가능 — 후속

### 7.1 `ConvoyState` 추가 필드 (03c §2.2 보강)

```
ConvoyState {
  // ... 기존 필드 ...
```

### 7.2 직렬화 형태 (JSON)

```json
"consumablesStash": [
  { "type": "MedKit", "count": 3 }
```

- enum은 문자열 직렬화 (Newtonsoft `StringEnumConverter`)
- 빈 인벤토리는 `[]` (null 아님, 항상 배열)

본 절은 **03c 보강 예고**. 03c v1.1 시점에는 본 필드 미정 → 08 확정 후 03c v1.2에서 정식 반영. 03c v1.1과 동시 적용은 의도적으로 미룸(분리 커밋).

## 8. 구현 가이드 (Dev 참조)

> `Crux-dev` 워크트리 구현자 참조용.

### 8.1 신규 타입

| 타입 | 네임스페이스 제안 | 성격 |
|---|---|---|
| `ConsumableType` | `Crux.Data` | enum, MedKit |
| `ConsumableDataSO` | `Crux.Data` | ScriptableObject — `type`·`displayName`·`description`·`iconSprite` |
| `ConsumableStack` | `Crux.Core` | POCO, 직렬화 대상 |
| `ConsumableInventory` | `Crux.Core` | `ConvoyInventory` 산하. `Add`·`Use`·`Count(type)` API |
| `MedKitEffect` | `Crux.Core.Effect` | static, `Apply(CrewMemberRuntime crew)` — 부상 단계 1단계 회복 |

### 8.2 통합 지점

- **`MaintenanceTicker`** — 정비 탭에서 응급 키트 사용 콜백. `ConsumableInventory.Use(MedKit)` → `MedKitEffect.Apply(crew)` → `injuryBattlesRemaining = 0` + `injuryState` 1단계 강등
- **`MarketController`** (신규) — 11 §3.3.1 기본 시장 화면에서 소모품 섹션 분리. 5개 묶음은 `inventory.Add(MedKit, 5)` + `funds -= 750`
- **`SaveManager`** (`docs/03c §9`) — `ConvoyState.consumablesStash` 직렬화 포함

### 8.3 효과 적용 순서

```
사용 버튼 클릭
  ↓
  ConsumableInventory.Has(MedKit) 체크
  ↓
  inventory.count -= 1
  ↓
  MedKitEffect.Apply(crew)  // injuryState/injuryBattlesRemaining 갱신
  ↓
  MaintenanceTicker.RefreshUI()  // 카드 갱신
  ↓
  RequestSave 트리거 없음 (정비 탭 조작은 03c §3.2 트리거 외)
```

다음 노드 이동 시점의 트리거(예: `BattleResultConfirmed` 다음 캠프 도착)에 의해 함께 영속화.

## 9. 오픈 이슈 / 결정 필요

1. **전투 중 소모품 사용 도입 시점** — Phase 2 후보. 도입 시 AP 비용·턴 종료 처리 등 추가 결정. MVP는 캠프 한정
2. **Phase 2 SKU 4종 우선순위** — RationFeast / RepairKit / MoraleBoost / FuelCache 중 첫 빌드 후 어느 것부터 도입할지. 사용자 결정 시점에 §4.2 표 갱신
3. **응급 키트 등급 분기** — "고급 응급 키트(2단계 회복)" 같은 등급 분기 도입 여부. MVP는 단일 등급, 등급 도입 시 가격·시장 분포 재설계 필요
4. **암시장 소모품 출현** — 11 §3.3.2 암시장 가중 추첨에 소모품 등급 포함 여부. MVP는 **파츠·완성 전차만**, 소모품은 기본 시장 한정
5. **튜토리얼 무상 지급** — 캠페인 인트로(스토리 도입부)에서 응급 키트 1~3개 무상 지급할지. 11 §3.3.1 가격 학습에 도움. 02 세계관 진행 시 결정
6. **5개 묶음 할인 도입** — MVP는 동일 단가. 첫 빌드 외부 피드백에서 "묶음 구매 인센티브 약함" 지적 시 도입

## 11. 범위 외

- **장식·도색·휘장**(`docs/13` 아트 콘셉트 후속) — 게임플레이 영향 없는 코스메틱은 본 문서 외
- **승무원 개인 장비**(헬멧·고글) — 04 트레잇 시스템과 중복. 도입 안 함
- **연구·기술 트리** — 1인 개발 범위 외, 시설 해금(`docs/11 §3.5`)으로 대체
- **소모품 제작·합성** — 시장 구매 단일 경로. 제작 시스템 도입 시 03b 정비 탭 확장 필요
- **임시 효과 버프 아이템** (전투 중 도핑 등) — Phase 2. AP·턴 시스템 깊은 영향
- **퀘스트 아이템** — 02 세계관 진행 시 별도 기획. 본 문서는 게임플레이 소모품만
