## 1. 설계 원칙

1. **정비는 확인 화면, 선택 화면이 아니다** — 대부분의 처리는 자동. 플레이어가 하는 능동 행동은 **각성 카드 선택**과 **파손 파츠 즉시 교체** 두 개뿐
2. **시간 틱은 "캠프 방문"** — 턴·전투 회수가 아닌 캠프(거점 노드) 진입 횟수를 기준. 휴식·대화 노드는 정비 시간 진행 없음 (`docs/09 §2.4`)
3. **자금 소모 없음** (MVP) — 정비는 무료. 자금은 상점·회식에만 소모 (`docs/11 §3.1`)
4. **파손 파츠는 장착 불가가 아니다** — 내구도에 비례한 성능 감쇠로 사용 가능. 전투 자체를 막지는 않음
5. **블라인드 규율** — 각성 카드 "언제"는 공개(큐 뱃지), "무엇이" 나올지는 선택 직전까지 비공개 (`docs/04 §3.4`)

## 2. 정비 씬 책임 범위

### 2.1 포함

- **파츠 내구도 복구** 진행 상황 확인
- **모듈 HP 복구** 진행 상황 확인
- **부상 승무원 복귀 예정** 표시 (`docs/09 §6`)
- **각성 카드 선택** (마크 레벨업 큐 해제)
- **파손 파츠 즉시 교체** (공용 인벤토리에서 새 파츠 선택)
- 정비 완료 이벤트에 따른 **사기 소폭 변동** (§5)

### 2.2 제외 (다른 화면 책임)

- 파츠 매각·구매 → **상점 탭** (`docs/11 §3.1`)
- 회식 (사기 +10~50) → **캠프 메뉴의 "회식" 액션** (`docs/11 §3.2`, docs/09 §2.3)
- 파츠 교체 **편성** 자체 → **편성 탭** (`docs/10 §2.2`)
- 시설 해금 → 자동 (`docs/09 §3`)

정비 탭은 **상태 확인 + 정비 관련 선택**에 한정한다.

## 3. 자동 처리 항목

### 3.1 파츠 내구도 복구

노획 파츠의 초기 내구도는 `docs/05b §5.3` 기준 (0.25~0.90 + 편차).

| 내구도 구간 | 전투 성능 계수 | 장착 가능 여부 | 비고 |
|---|---|---|---|
| 1.00 | ×1.00 | ✓ | 완전 상태 |
| 0.70~0.99 | ×1.00 | ✓ | 경미한 마모. 페널티 없음 |
| 0.40~0.69 | ×0.85 | ✓ | 성능 15% 감쇠 |
| 0.10~0.39 | ×0.65 | ✓ | 35% 감쇠 + 파손 아이콘 표시 |
| 0.00~0.09 | — | **✗** | **장착 불가** (수리 대기 or 교체) |

**회복 속도**: 캠프 방문 1회당 **+0.25** 내구도 회복 (상한 1.00).
- 0.05 상태 → 1 캠프 후 0.30 (장착 가능), 4 캠프 후 완전 회복
- 전투 중 자연 회복은 **없음** (수리병의 전투 중 모듈 수리는 별개, `docs/04 §1.5`)

**파손 파츠 `List<PartInstance>`는 `ConvoyInventory`의 주 풀에 그대로** — 별도 "수리 대기 풀"로 격리하지 않는다. 내구도가 0.10 미만이면 장착 UI에서만 필터링. 이유: 저장 구조 단순화.

### 3.2 모듈 HP 복구

전차 자체에 부착된 모듈 8종 (Engine/Barrel/MachineGun/AmmoRack/Loader/CaterpillarL/CaterpillarR/TurretRing). 각 모듈은 `Functional / Damaged / Destroyed` 3상태.

| 처리 시점 | 효과 |
|---|---|
| 전투 종료 직후 (자동) | `Damaged` → `Functional` 전환. **무료·즉시**. 이유: 경미 파손은 전투 간에 자동 정비 |
| 1차 캠프 방문 후 | `Destroyed` → `Damaged` |
| 2차 캠프 방문 후 | `Damaged` → `Functional` (완전 복구) |

**결과**: `Destroyed` 모듈이 `Functional`로 완전 복구되려면 **2 캠프 방문** 소요. 전투 중 발생한 `Damaged`는 다음 전투에서 즉시 쓸 수 있음.

### 3.3 부상 회복 (`docs/04 §8`·`docs/09 §6` 정합)

| 단계 | 조건 | 페널티 | 회복 조건 |
|---|---|---|---|
| 경상 (Minor) | 피격 관통 시 10% 확률 | 다음 전투 해당 승무원 능력 -10% | **1 전투 참전** 자동 해제 (출격·미출격 무관) |
| 중상 (Severe) | 관통 1% / 탄약고 유폭 100% | **2~3 전투 참전 불가** | 3 전투 경과(출격 카운트) 후 자동 `Minor`로 격하, 이후 1 전투 후 해제 |
| 치명상 (Fatal) | 스토리 이벤트만 | 영구 사망 | 회복 불가 |

**의사 합류 (Phase 2 스토리 보상)**:
- 중상 회복 단계 -1 전투 (3→2 전투)
- `Minor` → `None` 즉시 처리 (전투 경과 없이 캠프 복귀)
- 합류 시점은 `docs/02 세계관·스토리` 또는 `docs/16 일정`에서 확정 (현재 미정, §8 오픈 이슈)

**회복 진행 UI 표시**:
```
이리스 (기총/수리)  경상  복귀: 다음 전투 후
그린   (탄약수)     중상  복귀: 2 전투 후
```

## 4. 능동 선택 항목

### 4.1 마크 각성 카드 선택

> **스킬 풀 쿼리·요구조건·카드 데이터 모델**: [`04b_skill_system.md` §6](04b_skill_system.md#6-스킬-획득-파이프라인-각성-카드). 본 절은 정비 씬 UX·동선만 정의.

#### 4.1.1 큐 진입 조건

전투 종료 시 `04b §6` 파이프라인으로 승무원별 레벨업 이벤트가 생성:

```
for each crew in participatedCrew:
  for each axis in crew.class.axes:
    update killCounter / battleCounter
    if markThreshold 달성 (이중 게이트, docs/04 §3.3):
      queue.Add(MarkLevelUpEvent { crew, axis, newMarkLevel })
```

이벤트는 `ConvoyInventory` 또는 별도 `AwakeningQueue` 컨테이너에 누적된다.

#### 4.1.2 카드 수 분기

| 쿼리 결과 후보 수 | 처리 |
|---|---|
| 0개 | 스킵 (로그만 출력) |
| 1개 | **자동 획득** — 각성 팝업 표시 후 즉시 장착 제안 |
| 2~3개 | **카드 선택 UI** — 2~3장 가로 배치, 호버 설명 |
| 4개 이상 | 3장 랜덤 추출 후 카드 선택 (UI 폭 제약, 밸런싱) |

**중복 제거**: 이미 보유 중인 스킬·요구조건 불만족 스킬은 쿼리 결과에서 배제 (`docs/04 §4.5`).

#### 4.1.3 UI 동선

```
Hangar 진입 → "정비" 탭 뱃지에 "🔧 3" 표시 (큐 3건 대기)
→ 클릭 → 큐 순차 처리
   [승무원 이름] [축명] [Mark N 달성]
   ┌────────┐ ┌────────┐ ┌────────┐
   │ 카드 1 │ │ 카드 2 │ │ 카드 3 │
   └────────┘ └────────┘ └────────┘
   [호버 시 설명 툴팁 표시]
→ 선택 시 ownedSkills에 추가
→ 선택 직후 슬롯 장착 여부 확인 대화:
   "바로 장착하시겠습니까? (패시브 2/액티브 2 슬롯)"
   └ 예 → 장착 UI 호출 (편성 탭 열기)
   └ 아니오 → 보유 목록에만 추가
→ 다음 큐 항목 처리 (없으면 큐 종료)
```

**큐 처리는 순차**. 한 번에 한 승무원만. 이유: 여러 카드 동시 노출 시 선택 피로 증가 + UI 복잡.

#### 4.1.4 취소·뒤로가기 정책

- 카드 선택 UI에서 **뒤로가기 없음** — 큐 항목은 진입하면 반드시 하나 선택해야 종료
- 스킵 버튼 없음 — 이유: "획득 기회를 날려도 되는" 선택지는 플레이어 경험을 나쁘게 만듬
- 예외: 후보 4+개 중 랜덤 3장이 추출된 경우에도 "재뽑기" 없음. 랜덤은 초기에 한 번만

### 4.2 즉시 교체 (파손 파츠 대체)

내구도 <0.10 또는 `Destroyed` 상태로 장착 중인 파츠를 수리 대기 없이 새 파츠로 교체하는 단축 경로.

#### 4.2.1 진입

정비 탭에서 각 전차 슬롯 상단에 **파손 배지** 표시:
```
[로시난테]
  ⚠ 파손 파츠 2개
    · 주포: 76mm 단포신 (내구도 0.05)  [교체]
    · 엔진: V8 디젤 (내구도 0.00)       [교체]
```

#### 4.2.2 교체 흐름

`[교체]` 버튼 → 공용 인벤토리에서 **같은 카테고리 + 호환 가능 + 장착 가능 내구도(≥0.10)** 파츠 리스트 팝업:
- 리스트 선택 → 즉시 장착 + 기존 파손 파츠 인벤토리 회수
- "교체할 파츠 없음" 메시지 → 수리 대기만 가능
- 호환성 체크는 `docs/05 §3`의 3중 체계 (하중·출력·규격) 그대로 적용

#### 4.2.3 편성 탭과의 관계

편성 탭에서도 파츠 교체는 가능 (`docs/10 §2.3`). **정비 탭의 즉시 교체는 파손 파츠에 한정된 단축 경로**. 일반 교체·교환은 편성 탭에서.

## 5. 정비 씬 내 사기 이벤트

정비 탭 내에서 자동 발생하는 소폭 사기 변동. 능동 이벤트(회식)는 캠프 메뉴 소관.

| 트리거 | 사기 변동 | 대상 |
|---|---|---|
| 파손 파츠 완전 복구 (내구도 1.0 도달) | +2 | 해당 전차 전 승무원 |
| 모듈 `Destroyed → Functional` | +3 | 해당 전차 전 승무원 |
| 부상 해제 (Minor → None) | +3 | 해당 승무원 + 동승 4인 |
| 부상 해제 (Severe → Minor) | +5 | 해당 승무원 + 동승 4인 |
| 마크 각성 카드 획득 | +5 | 해당 승무원만 |

**처리 타이밍**: 캠프 진입 직후 일괄 적용. 정비 탭 클릭 시점이 아니라 **노드 도착 시점**.

**로그**:
```
[CRUX] [MAINT] 캠프 도착 정비 처리:
   로시난테: 주포 수리 완료 (+2 사기)
   이리스: 경상 해제 (+3 사기)
```

**특수**: 전사자 발생 후 첫 캠프 방문 시 "추모식" 자동 발동 (`docs/09 §2.3`). 추모식은 **캠프 수동형 이벤트**로 정비 범위 밖이지만, 같은 타이밍에 실행되므로 구현 시 순서 조정 필요 (§7.3 참조).

### 6.2 중앙 작업공간 레이아웃 (docs/10 §2.1 3분할 재사용)

정비 탭 클릭 시 중앙 영역은 **세 섹션 스택**:

```
┌──────────────────────────────────────────────────────────────┐
│  🔧 정비 (Maintenance)                   캠프 방문: 4회차     │
├──────────────────────────────────────────────────────────────┤
│  ▼ 각성 대기 (2건)                                    [처리] │
│   · 프리테나 (조종수) Mark 2 달성                             │
│   · 리리드  (포수)   Mark 3 달성                             │
├──────────────────────────────────────────────────────────────┤
│  ▼ 전차 정비 현황                                             │
│   로시난테                                                    │
│     · 주포    내구도 0.05  파손  [교체]                       │
│     · 엔진    내구도 0.42  캠프 2회 후 완전 복구              │
│     · 캐터필러 Destroyed   캠프 1회 후 Damaged                │
│                                                               │
│   셔먼 (보관)                                                 │
│     · (모든 모듈 Functional, 내구도 전체 1.0)                 │
├──────────────────────────────────────────────────────────────┤
│  ▼ 승무원 회복                                                │
│   · 이리스  경상  다음 전투 후 해제                           │
│   · 그린    중상  2 전투 후 경상                              │
└──────────────────────────────────────────────────────────────┘
```

### 6.3 각 섹션 상세

#### 6.3.1 각성 대기 섹션
- 큐가 **비어 있으면 섹션 자체 숨김** (UI 단순화)
- 큐가 있으면 **큐 수량 뱃지 + "[처리]" 버튼** 상단 표시
- "[처리]" 클릭 → 카드 선택 모달 진입 (§4.1.3)
- 큐 처리 중 뒤로가기 방지

#### 6.3.2 전차 정비 현황
- 부대 전 전차 (출격·보관 모두)를 세로 리스트
- 각 전차마다 파손 파츠 + 손상 모듈만 나열. 정상은 생략 (또는 "모두 정상" 한 줄)
- `[교체]` 버튼은 파손 파츠에만 표시 (§4.2.1)

#### 6.3.3 승무원 회복
- 부상 상태 승무원만 리스트 (`None` 제외)
- 복귀 예정 턴 공개

### 6.4 정비 탭 뱃지

Hangar 좌 메뉴의 정비 탭에 **처리 대기 뱃지** 표시:

```
🔧 정비  ●3    ← 각성 카드 큐 3건
```

뱃지 기준:
- 각성 카드 큐 항목 수 (파손 파츠·부상은 카운트 제외 — 플레이어 압박 줄임)
- 큐가 있으면 노란색, 없으면 회색

## 7. 구현 책임 분리 (Dev 가이드)

> 본 절은 `Crux-dev` 워크트리 구현자 참조용. 네임스페이스·클래스명은 제안.

### 7.1 신규 타입

| 타입 | 네임스페이스 제안 | 성격 |
|---|---|---|
| `MaintenanceState` | `Crux.Core.Maintenance` | 부대 정비 상태 집계 (각성 큐·파손 목록·부상 목록) |
| `MarkLevelUpEvent` | `Crux.Data.Crew` | 각성 큐 항목 (승무원·축·새 마크 레벨) |
| `AwakeningQueue` | `Crux.Core.Maintenance` | `List<MarkLevelUpEvent>` + 진입·이탈 API |
| `MaintenanceTickerSO` | `Crux.Data.Maintenance` | 캠프 방문 진행 틱 계수 (+0.25 내구도, 2 캠프 복구 등 튜닝값) |

### 7.2 서비스 계층

| 함수 | 책임 |
|---|---|
| `MaintenanceTicker.OnCampVisit(convoy)` | 캠프 진입 시 호출. §3 자동 처리 일괄 수행 |
| `MaintenanceTicker.OnBattleEnd(convoy)` | 전투 종료 시 호출. 모듈 `Damaged→Functional` + 각성 큐 채움 + 부상 전투 카운트 증가 |
| `SkillPoolQuery.Query(crew, axis)` | `docs/04 §4.5` 파이프라인. 후보 스킬 리스트 반환 |
| `DurabilityUtil.GetPerformanceMultiplier(durability)` | §3.1 성능 계수 룩업 |

### 7.3 통합 지점

- **`BattleController`** — 전투 종료(Victory/GameOver) 직후 `MaintenanceTicker.OnBattleEnd` 호출
- **`WorldMapController`** (신규 or 기존) — 거점 노드 도착 시 `MaintenanceTicker.OnCampVisit` 호출
- **`HangarTabController`** — 정비 탭 진입 시 `MaintenanceState` 구독, UI 바인딩
- **`MissionCompleteModalBinder`** — 전투 종료 모달에 "각성 큐 N건 대기" 안내 라인 추가 가능 (선택)

### 7.4 추모식 실행 순서 (§5 특수)

```
캠프 진입:
  1. 전사자 확인 (직전 전투에서 Fatal 발생 시)
  2. IF 전사자 있음:
       추모식 이벤트 발동 → 사기 조정 → UI 블로킹 팝업
  3. MaintenanceTicker.OnCampVisit  ← 추모식 후
  4. UI 정비 탭 정상 진입 가능
```

추모식이 정비 처리보다 먼저. 이유: "정비 이벤트 +5" 이전에 추모 서사가 우선돼야 감정 곡선이 맞음.

## 8. 오픈 이슈 / 결정 필요

1. **의사 캐릭터 합류 시점** — `docs/09 §6` 언급만. 구체 노드·스토리 트리거 미정. `docs/02 세계관`·`docs/16 일정`과 동시 확정 필요
2. **내구도 0.40~0.69 구간 성능 계수(×0.85)** — 초안. 플레이테스트에서 파손 파츠가 "여전히 쓸 만한지" 검증 필요
3. **각성 카드 4+ 추출 상한 3장** — 후보 풀이 실제로 4+가 되는 케이스가 얼마나 잦은지 측정 필요. 매우 드물면 규칙 단순화 가능
4. **파손 파츠 "즉시 교체" UI가 편성 탭과 UX 일관되는지** — `docs/10 §2.3` 파츠 인벤토리 오버레이와 동일 컴포넌트 재사용 가능? 실제 빌드 후 검증
5. **추모식 스킵 가능 여부** — `docs/09 §2.3` "스킵 불가" 명시. 본 문서는 그 규칙 따름. 변경 시 양쪽 수정
6. **캠프 방문 카운트 저장** — 세이브 스키마에 `campVisits: int` 필드 추가 필요. `docs/11 경제`·세이브 기획(`docs/...` 미작성)에 반영 필요

## 9. 튜닝 훅

| 항목 | 초안 | 튜닝 사유 발생 시 |
|---|---|---|
| 내구도 회복 +0.25/캠프 | 0.25 | 파손 파츠가 2~3 캠프 안에 복귀하는 체감 맞는지 |
