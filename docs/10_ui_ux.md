## 0. 개요

### 0.1 Unity UI 시스템 선택

- **신규 UI는 uGUI (Unity UI / Canvas 기반)**로 작성한다.
- 현재 전투 중 HUD는 `OnGUI`(IMGUI)로 구현됨 → **당분간 유지**, 전투 기능 안정화 이후 uGUI로 점진 이관 (TD-07 로 원장에 추가).
- UI Toolkit(UXML/USS)은 러닝커브·자료 부족으로 **채택하지 않음**.
- OnGUI와 uGUI는 동일 씬에서 **공존 가능** — 이관 중에도 기능 손상 없음.

### 0.2 UI 계층 원칙

- **한 씬에 Canvas 2~3개 이내**: `MainCanvas` (월드 UI 오버레이) / `OverlayCanvas` (팝업·툴팁) / `DebugCanvas` (F1 디버그).
- **패널은 프리팹으로 독립**: `HangarPanel.prefab`·`InventoryPanel.prefab` 등. 씬에 하드코드 배치 금지.
- **데이터 바인딩은 C# 컴포넌트가 담당** — `HangarUI` MonoBehaviour 가 `TankInstance`·`ConvoyInventory` 상태 읽고 UI 갱신.

### 0.3 첫 빌드 범위 (1~2개월)

- 최소형 5화면: **타이틀 · 월드맵 · Hangar(편성 중심) · 전투 · 전투 결과**.
- 대화·휴식·상점·회식·정비·NPC 클릭은 **이후 기능**. 문서에 뼈대만 남기고 내부 디테일은 작성 시점에 보강.

## 1. 씬 맵

```
[Title]
   ↓ 신작/이어하기
[WorldMap] ←─────────────────────┐
   │                             │
   ├─ 전투 노드 → [Battle] → [Result] ─┐
   ├─ 대화 노드 → [Dialog] ─────────────┤
   ├─ 거점 노드 → [Hangar(Camp)] ────────┤
   └─ 휴식 노드 → [Rest 팝업] ─────────┘
```

| 씬 | 첫 빌드 구현 여부 | 비고 |
|---|---|---|
| Title | ✅ 최소형 | 신작/이어하기/종료 |
| WorldMap | ✅ 최소형 | 5~10 노드 일직선 |
| Hangar (Camp 통합) | ✅ 편성 탭만 | 나머지 탭 비활성 |
| Battle (TacticalScene) | ✅ 이미 존재 | OnGUI HUD 유지 |
| Result (전투 결과) | ✅ 최소형 | 격파·노획·자금만 |
| Dialog | ⏸ 이후 | VN 프레임 |
| Rest | ⏸ 이후 | 팝업 1장 |

Unity Scene 파일 수는 **5개로 고정** (Title·WorldMap·Battle·Hangar·Dialog). Result/Rest 는 별도 씬 아닌 WorldMap 위 **오버레이 Canvas**로 처리 (씬 전환 비용 절감).

## 2. Hangar (캠프) 와이어프레임

> **첫 빌드 최우선 코드 타깃**. 편성 탭만 구현, 나머지 탭은 UI 골격만 배치하고 잠금 표시.

### 2.1 전체 레이아웃

```
┌───────────────────────────────────────────────────────────────────┐
│  [자금: ₩12,340]   [사기: 85]              [월드맵으로 돌아가기]   │  ← 상단 바
├─────────┬───────────────────────────────────────┬─────────────────┤
│ 편성  ●│                                        │ ▸ 선택된 전차   │
│ 정비   │                                        │   · 로시난테    │
│ 상점   │          중앙 작업공간                 │   · HP 200/200  │
│ 스킬   │       (선택된 탭의 내용)               │   · 장갑 중형   │
│ 회식   │                                        │ ▸ Trait         │
│ 인물   │                                        │   · 노련한 포수 │
│        │                                        │ ▸ 승무원 5      │
│        │                                        │   · 포수 / 조종 │
│        │                                        │   · 장전/관측/통│
│        │                                        │                 │
│        │                                        │ [출격 편성 토글]│
└─────────┴───────────────────────────────────────┴─────────────────┘
  좌 메뉴 (160px)        중앙 (flex)                우 패널 (280px)
```

- **좌 메뉴**: 6 탭 (편성·정비·상점·스킬·회식·인물). 첫 빌드에서 **편성만 활성**, 나머지는 잠금 아이콘.
- **중앙**: 선택된 탭의 작업 공간.
- **우 패널**: 현재 선택된 객체(전차/파츠/NPC)의 상세 정보. 항상 표시.

### 2.2 편성 탭 — 중앙 작업공간

```
┌─────────────────────────────────────────────────────────────┐
│  출격 슬롯 (3/5)   * 초기 단계: 3, 확장 후 5                │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐             │
│  │ 로시 │ │ T-34 │ │ 없음 │ │ 🔒  │ │ 🔒  │ ← 잠김       │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘             │
│                                                             │
│  보관 (5/5)                                                 │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐             │
│  │ 셔먼 │ │ 판터 │ │ 없음 │ │ 없음 │ │ 없음 │             │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘             │
│                                                             │
│  [파츠 인벤토리 열기]   [호환성 검사]                       │
└─────────────────────────────────────────────────────────────┘
```

- **슬롯 클릭** → 우 패널에 해당 전차 정보 + 파츠 장착 상태 표시.
- **슬롯 드래그** → 출격 ↔ 보관 위치 교체.
- **빈 슬롯 클릭** → 보관 목록에서 전차 선택 팝업.
- **[출격 편성 토글]** (우 패널 버튼) → 선택된 전차를 출격 슬롯으로 이동/제거.

### 2.3 편성 탭 — 파츠 인벤토리 서브뷰

**[파츠 인벤토리 열기]** 버튼 클릭 시 중앙에 오버레이 패널로 확장:

```
┌─────────────────────────────────────────────────────────────┐
│  로시난테 ▼  (대상 전차 선택)          [X 닫기]              │
├─────────────────────────────────────────────────────────────┤
│  차체 슬롯 (카테고리별)                                     │
│  ┌──────────┐ 주포:   [ 76mm 장포신 ]    [교체]            │
│  │ 로시난테 │ 장갑:   [ 중장갑판 ]      [교체]            │
│  │  모형    │ 엔진:   [ V8 디젤 ]       [교체]            │
│  │          │ 궤도:   [ 표준궤 ]        [교체]            │
│  └──────────┘ 보조:   [ 연막탄 발사기 ]  [교체]            │
│                                                             │
│  선택 슬롯: 주포                                            │
│  보유 파츠 리스트 (호환만 표시 ☑)                           │
│  ┌─ 76mm 장포신   ▣ 장착중   관통 85   호환 ✓           │
│  ├─ 88mm 단포신   ◌ 보관      관통 110  호환 ✗ (터렛 소형) │
│  ├─ 50mm 속사포   ◌ 보관      관통 50   호환 ✓           │
│  └─ [합성] 50mm + 50mm → 76mm 개량  ← P4-e 진입점         │
└─────────────────────────────────────────────────────────────┘
```

- **호환성 표시**: 각 파츠 옆 ✓/✗ 아이콘. 호버 시 툴팁으로 실패 사유 표시 (터렛 크기·중량·슬롯 수 등).
- **호환만 보기 ☑ 체크박스**: 기본 ON. 해제 시 전 파츠 표시.
- **[합성] 항목 (P4-e)**: 현재 슬롯에 같은 카테고리 파츠 2개를 드래그하면 상위 등급 합성 후보 자동 노출. 합성 재료 소모 → 결과 파츠 인벤토리 추가 후 장착 가능. **합성 UI는 별도 팝업 아닌 인벤토리 리스트에 자연 통합**.
- **[교체] 버튼** → 우측 파츠 리스트 하이라이트. 파츠 선택 → 즉시 장착 (점진 장착 허용, 미장착 슬롯 보유 가능).

### 2.4 편성 탭 — Trait·승무원 프리뷰 (우 패널)

```
▸ Trait (전차 고유 특성)
  · 노련한 포수     명중 +5
  · 튼튼한 궤도     기동 시 사기 감소 감소
▸ 승무원 5
  · 포수   김철수   마크 2 (명중 +2)
  · 조종   박영희   마크 1 (회피 +1)
  · 장전   이민수   마크 0
  · 관측   최수정   마크 1 (시야 +1)
  · 통신   정대훈   마크 0
```

- **클릭**: 승무원 한 명 클릭 시 하단에 스킬 큐 표시 (쌓인 마크 + 각성 대기 스킬 카드). 첫 빌드에서는 **조회만**, 선택은 스킬 탭에서.
- **Trait 호버**: 구체 수치 툴팁.

### 2.5 나머지 탭 (첫 빌드 잠금 — UI 골격만)

| 탭 | 첫 빌드 상태 | 기능 설명 (이후) |
|---|---|---|
| 정비 | ✅ 활성 | 차체 수리·모듈 회복·부상 회복·마크 각성 카드 — 상세 [`03b_maintenance.md`](03b_maintenance.md) |
| 상점 | 🔒 잠금 | 파츠 매입·매각·탄약 보급 |
| 스킬 | 🔒 잠금 | (정비 탭에 흡수 예정 — 03b §4.1 각성 카드와 중복) |
| 회식 | 🔒 잠금 | 사기 +10~20 이벤트 (자금 소모) |
| 인물 | 🔒 잠금 | NPC 목록 · 관계도 · 대화 진입 |

**정비 탭 활성화 (2026-04-21)**: 스킬 각성 카드 UX가 정비 흐름에 자연 통합되어 "스킬" 탭은 별도 진입점 없이 정비 탭 내에서 처리. `docs/10 §2.5`의 스킬 탭은 후속 제거 검토 대상.

잠금 탭 클릭 시 "곧 출시" 토스트 메시지 표시. 활성화 시점은 캠페인 단계별 해금 (`docs/09 §3`).

## 3. 전투 결과 화면

전투 씬 종료 직후 월드맵 씬 위에 **풀스크린 오버레이 Canvas** 표시.

```
┌─────────────────────────────────────────────────────────────┐
│  ▼ 작전 완료: 섬멸                                          │
├─────────────────────────────────────────────────────────────┤
│  격파: 적 전차 4                                            │
│  아군 손실: 0 (부상 1)                                      │
│  사기 변화: +8  (85 → 93)                                   │
├─────────────────────────────────────────────────────────────┤
│  노획 파츠 3종                                              │
│  · 76mm 장포신  (보관됨)                                    │
│  · 중장갑판     (보관됨)                                    │
│  · V6 가솔린    (보관됨)                                    │
├─────────────────────────────────────────────────────────────┤
│  자금 보상: +₩2,500                                         │
│                                                             │
│             [확인]                                          │
└─────────────────────────────────────────────────────────────┘
```

- **1장 팝업**. 섹션별 페이드인 애니메이션(0.3s씩).
- **[확인]** → 오버레이 닫고 월드맵 커서가 다음 노드로 이동.
- **첫 빌드 단순화**: 슬라이드·보이스 없음. 순수 텍스트 + 아이콘.

## 4. 월드맵 화면

```
┌─────────────────────────────────────────────────────────────┐
│  [자금: ₩12,340]   [사기: 93]              [캠프 메뉴 (Tab)]│
├─────────────────────────────────────────────────────────────┤
│                                                             │
│      ●━━━●━━━●━━━◉━━━○━━━○━━━○                              │
│   시작  전투  휴식  거점  전투  대화  전투 ...              │
│                  ▲                                          │
│                 현재                                        │
│                                                             │
│   (일러스트 또는 단색 배경 + 노드 그래프)                   │
│                                                             │
│             [다음 노드 진입]                                │
└─────────────────────────────────────────────────────────────┘
```

- **노드 표시**: 완료 ●  현재 ◉  미개방 ○.
- **노드 종류 아이콘**: 전투(⚔) 대화(💬) 거점(🏕) 휴식(🌙). 첫 빌드는 텍스트 레이블로 갈음 가능.
- **[다음 노드 진입]** 클릭 → 노드 종류에 따라 해당 씬/팝업 로드.
- **일직선 그래프**: 좌→우 한 줄. 분기 없음.
- **첫 빌드**: 단색 배경 + 점-선 그래프만. 지도 일러스트는 이후.

## 5. 전투 HUD 개편 방향

- **현재**: `BattleHUD.cs` (OnGUI 기반). 기능 전부 작동 중.
- **첫 빌드 동안**: 유지. uGUI 이관하지 않는다.
- **이관 시점**: 전투 기능 안정화 후 (목표 마일스톤: 첫 빌드 완성 이후 리팩토링 슬롯에서).
- **이관 범위**: 하단 유닛 패널·상단 배너·사격 프리뷰·반응 경고. 디버그 오버레이(F1)는 OnGUI 유지 가능 (에디터 툴 성격).
- **신규 TD-07 등록** (§10 TD 원장에): `BattleHUD OnGUI → uGUI 이관`.

## 8. UI 스타일 톤

> **뼈대 결정만**. 구체 색·폰트·아이콘은 미확정, 아트 패스 시점에 결정.

### 8.1 확정 방향

- **다크 배경 + 금속·바랜 노랑 액센트** (포스트아포·전쟁 유물 톤).
- **폰트**: Sans-serif 2종 (제목 굵게 · 본문 기본). 픽셀 폰트는 채택 안 함 (전차 스프라이트는 픽셀이지만 UI는 가독 우선).
- **아이콘**: 모노크롬 벡터 (교체 용이). 컬러 아이콘은 상태 표시용 제한 사용 (사기·자금·경고).

### 8.2 미확정 (아트 패스 때 결정)

- 구체 HEX 색상표
- 폰트 선정 (상용·오픈소스)
- 버튼·패널 9-slice 스타일
- 애니메이션 곡선·타이밍
