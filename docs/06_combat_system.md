# 6. 전투 시스템

> 개정: 2026-04-15 — 실시간 서바이벌 섹션 제거. 턴제 Hex 전술이 유일한 전투 방식으로 확정.
> 적 AI·의사결정 상세는 `docs/12_enemy_ai.md`가 권위 문서.

## 1. 전투 방식

**턴제 Hex 그리드 전술**. 부대(1~6대 전차) 규모의 소규모 교전 단위 전투.

- **아군 턴 → 적 턴** 순환 (docs/04 §7 하이브리드 C AP 경제)
- 전투 시작 시 **이니셔티브 굴림**으로 선공 그룹 결정 (docs/03 §2.3)
- 턴 중 **반응 사격·오버워치**로 상대 턴에 개입 가능 (미니 이니셔티브)
- 승리/패배 조건은 스테이지별 스크립트 (docs/03 §3.4)

### 1.1 씬 구조

| 씬 | 역할 |
|---|---|
| `StrategyScene` | 8×10 표준 전술 전투 — 메인 플레이 |
| `TerrainTestScene` | 12×12 지형 테스트 + 디버그 오버레이 — 기능 개발용 |
| `FireActionScene` | 주포·MG 사격 시네마틱 연출 |

## 2. 그리드 구조 — 육각형 (Hex)

- **타일 형태**: flat-top hex (CRUX 프로젝트 기준)
- **좌표계**: odd-q offset — `HexCoord` 타입이 변환 유틸 제공
- **이웃 수**: 6방향 (60° 간격)
- **거리 함수**: 육각 거리 `(|dq| + |dr| + |dq+dr|) / 2`
- **이동 비용**: 1 셀당 정해진 AP — 모든 방향 동일 (대각선 파라독스 없음)

### 2.1 왜 육각인가

| 기준 | 근거 |
|---|---|
| 기하학적 정직성 | 모든 6방향이 진짜 등거리 — 사각+8방의 Chebyshev 왜곡 없음 |
| 이동 범위 | 원형에 가깝게 퍼짐 — 플레이어 직관과 일치 |
| 측면 공격 명료성 | 6구역 장갑이 타일 변 6개와 1:1 매칭 |
| 엄폐 일치성 | **6 방호면 = 6 타일 변** — 데이터·시각·물리 동일 언어 |
| 장르 정합성 | Panzer General·Steel Panthers 등 전차 전투 게임 역사적 표준 |
| 수학 성숙도 | Axial/Cube 좌표 체계가 업계 표준, 레퍼런스 풍부 |

2026-04-13 결정. 초기 단계 비가역 구조 결정 원칙에 따라 고정.

## 3. 턴 구조

### 3.1 기본 흐름

```
전투 시작
  → 이니셔티브 굴림 (전차별 React + Morale/5 + 특성 + 차체 속도 + d20)
  → 선공 그룹 결정
  → 반복:
      아군 턴 — 모든 아군 유닛 순차 (순서 자유)
      적 턴   — 모든 적 유닛 순차 (AI 결정, docs/12)
  → 종료 조건 도달 시 탈출 (승리/패배/후퇴)
```

### 3.2 AP 경제 — 하이브리드 C

- **전차 공용 AP 8** — 이동·기본 사격·재장전 여기서 소모
- **승무원별 스킬 쿨다운** — 스킬은 대부분 쿨다운(1~5턴)으로 제약
- **일부 강력 스킬만 추가 AP** — 듀얼(+5 AP), 전탄 발사(전량) 등

상세: docs/04 §7

### 3.3 주포 구경 계층 (Fire AP)

**주포 AP는 구경별 차등**. 재장전 시간은 사격 AP에 녹임(별도 재장전 턴 없음).

| 계층 | 구경 (예시) | Fire AP | Overwatch AP (Fire×2) | 비고 |
|---|---|---|---|---|
| **S** 소구경 | 20~40mm 기관포 | 2 | 4 | 경차량·드론 주무기 |
| **M** 표준 | 75~90mm 전차포 | 3 | 6 | 중형 전차 기본 |
| **L** 대구경 | 100~122mm | 4 | 8 | 중전차 — OW 사실상 한계 |
| **XL** 중포 | 152mm+ | 5 | 10 | 중전차·돌격포 — OW 불가 |
| **SPG** 곡사 | 자주포 155mm+ | 6 | 12 | 간접사격 전문 — OW 불가 |

**병종별 MaxAP**:

| 병종 | MaxAP | 기본 주포 계층 | 턴당 사격 기대 |
|---|---|---|---|
| Light 경차량 | 8 | S (AP 2) | 이동 + 사격 3~4회 |
| Medium 중형 | 6 | M (AP 3) | 이동 2 + 사격 1 |
| Heavy 중전차 | 5 | L (AP 4) | 정지 시 1회, 이동 시 0 |
| SPG 자주포 | 4 | XL/SPG (AP 5~6) | 2턴 1발 페이싱 |
| Drone 드론 | 10 | S (AP 2) | 고기동 + 다수 사격 |

**구현**: `TankDataSO.mainGunFireAP` 필드로 구경별 지정. `GameConstants.FireCost` 폐기. `GetOverwatchCost(fireAP) = fireAP × 2` 함수화.

### 3.4 자동 상호교전 (Auto Counter-Attack)

**기본 규칙**: 피격자가 생존 + 조건 충족 시 **자동 반격 1회**.

**반격 성립 7조건** (모두 만족):

1. 피격자 생존 (HP>0, 모듈 치명 파괴 아님)
2. 피격자 AP ≥ 반격 주포 Fire AP
3. 피격자 주포 탄약 ≥ 1
4. 공격자가 피격자 사거리·LOS 안
5. 공격자가 피격자의 **전방 호(±60°)** 안 (측·후면은 반격 불가, 크루 스킬로 우회)
6. 피격자가 **오버워치·반격 면역 상태 아님**
7. 피격자가 **이동 상태 아님** (이동 중 피격 = 적 오버워치. 오버워치는 반격 면제이므로 해당 없음)

**비용**: Fire AP + 탄약 1발 (일반 사격과 동일)

**명중률 페널티**: **−15%** (크루 스킬 "냉정 조준"으로 상쇄 가능)

**연쇄 반격 차단**: 공격 1회당 반격 1회. 반격의 반격 없음 (크루 스킬로만 가능)

### 3.5 이니셔티브 — 선제 반격

**규칙**: 교전 순간 공격자·피격자 **현재 AP 비교**.

- **피격자 AP < 공격자 AP** → 공격 해결 후 일반 반격
- **피격자 AP ≥ 공격자 AP** → **선제 반격** (공격자가 쏘기 전에 피격자 먼저 발사)
  - 선제 반격으로 공격자 **격파·주포 파괴** 시 원 공격 무효 (반격이 공격을 끊음)
- **크루 스킬 "직감" 발동** → AP 비교 무시하고 강제 선제

### 3.6 오버워치 재정의

- **AP 비용**: 주포 Fire AP × 2 (L/XL/SPG급은 사실상 불가)
- **반격 면역**: 오버워치 중인 유닛을 공격해도 자동 반격 발생하지 않음 (AP를 이미 선지불했으므로)
- **이동 중 피격자 처리**: 적 오버워치에 당한 것이므로 반격 미성립 (§3.4 조건 7)
- 트리거 타입: MoveTrigger / FireTrigger / SightTrigger (docs/04 §4.1)
- 유닛당 1턴 당 반응 발동 **1회**
- 시각 연출: `FireActionScene` 전환 또는 `ReactionFireSequence` 6박자 (공격자 기준)

### 3.7 크루 스킬 (반격 관련)

**공통 원칙**: 모든 크루 스킬은 **조건 만족 시 확률 발동**. **사기(Morale)가 발동률에 영향**.

**사기 보정 공식**: `실제 발동률 = 기본률 × (0.5 + Morale/200)`
- Morale 0 → 0.5×, Morale 100 → 1.0×, Morale 200(극한) → 1.5×
- 패닉 상태(Morale < 20) → 강제 0

| 스킬 | 보유 포지션 | 조건 | 효과 | 기본 발동률 |
|---|---|---|---|---|
| **직감** Intuition | 전차장/포수 | 피격 직전 | AP 비교 무시, 강제 선제 반격 | 25% |
| **측후면 경계** Peripheral Watch | 전차장/장전수 | 측·후면 피격 | 전방 호 제한 1회 무시 | 20% |
| **연쇄 반격** Follow-Up | 포수 | 반격 직후 재피격 | 추가 반격 1회 (AP·탄약 별도) | 15% |
| **냉정 조준** Cold Aim | 포수 | 반격 실행 시 | −15% 명중 페널티 상쇄 | 30% |
| **속장** Quick Reload | 장전수 | 반격 실행 시 | 반격 Fire AP −1 (최소 1) | 20% |
| **위기 대응** Crisis Reflex | 전차장 | Morale ≤ 40 피격 | "직감" 발동률 +50%p (이 턴 한정) | 패시브 |

**전제**: 현재 `CrewMember`에 `Morale` 필드 없음 — 크루 스킬 구현 전 필드 신설 필요. 1단계 구현(반격 코어)은 스킬 없이도 작동하도록 분리.

## 4. 관통·방호 시스템

### 4.1 입사각과 유효 장갑

```
유효 장갑 = 기본 장갑 / cos(입사각)
입사각 70° 이상 → 자동 도탄
```

- 입사각은 공격자→대상 벡터와 대상 차체의 피격면 법선의 각도
- 도탄 시: 파편 파티클 + "땡!" 효과음
- 관통 시: 내부 데미지 + 모듈 손상 판정

### 4.2 6섹터 장갑

전차 차체 각도(헐 방향) 기준 6등분:

| 구역 | 각도 범위 | 의미 | 일반 장갑 |
|---|---|---|---|
| **Front** | 0° ± 30° | 전면 | 가장 두꺼움 |
| **FrontRight** | 60° ± 30° | 우전 (측면 장갑의 강한 쪽) | 두꺼움 |
| **RearRight** | 120° ± 30° | 우후 | 얇음 |
| **Rear** | 180° ± 30° | 후면 | 가장 얇음 |
| **RearLeft** | 240° ± 30° | 좌후 | 얇음 |
| **FrontLeft** | 300° ± 30° | 좌전 (측면 장갑의 강한 쪽) | 두꺼움 |

**플랭킹 전술성**: 차체를 정면으로 돌리지 못한 상태에서 사격 받으면 즉시 측·후면 노출. 포탑 회전과 차체 회전을 독립적으로 관리해야 하는 이유.

### 4.3 모듈 피격

| 모듈 | 파손 효과 |
|---|---|
| **캐터필러** | 이동 AP 2배 (docs/05 §2.6) |
| **엔진** | 최대 이동 -30%, 과열 위험 증가 |
| **포탑 구동** | 회전 AP 2배 |
| **탄약고** | 유폭 판정 — HP 즉사 또는 대량 손실 |
| **주포** | 명중·데미지 -30% |
| **기총** | 기총사수 버스트 -2, 명중 -10% |

**전투 중 복구**: 기총사수/수리병의 수리 스킬로 일부 모듈 복구 가능. 수리 대상 모듈별 마크 숙련도 적용 (docs/04 §1.5).

**피격 누적과 노획**: 격파 당시 모듈별 누적 피격이 노획 확률에 영향 (docs/05 §5 참조).

## 5. 엄폐 시스템 — 6면 방호 슬롯

### 5.1 변(edge) 기반 엄폐

- **엄폐물은 셀이 아닌 "변(edge)"** 에 속성 부여
- 각 셀은 최대 6개의 방호면 플래그를 가짐: `N / NE / SE / S / SW / NW`
- 공격자→대상 방향을 **6방향 슬롯에 스냅** 후, 해당 슬롯이 방호 플래그에 포함되면 엄폐 유효
- 엄폐율(`CoverRate`)은 HP에 비례
- **HP 감소 시 방호면 슬롯이 순차적으로 소실** (예: 완전 상태 3면 → HP 50% 2면 → HP 25% 1면)

```
예: 북쪽과 북동쪽을 막는 잔해 더미
protectedFacets = HexFacet.N | HexFacet.NE
→ 북에서 온 공격: 엄폐 유효
→ 북동에서 온 공격: 엄폐 유효
→ 동에서 온 공격: 엄폐 무효 (우회 성공)
```

### 5.2 방호면 수 제한

| 엄폐물 크기 | 최대 보호면 |
|---|---|
| Small | 1면 |
| Medium | 3면 |
| Large | 3면 |

게임플레이 데이터 레이어에서 강제. 3면을 초과하는 "요새급" 엄폐물은 구조물 카테고리로 별도 관리.

### 5.3 우회 전술성

대각 한 칸 이동만으로 방호면을 벗어날 수 있어, 이동 AP 소비로 엄폐 무효화가 가능 → 이동 결정이 명확한 트레이드오프를 만듦.

## 6. 유닛 회전 체계

- **헐 각도**: 60° 단위로 스냅 (6 방향)
- **포탑 각도**: 연속 각도 (차체와 독립 회전)
- 이동 시 차체가 이동 방향으로 자동 회전
- 정지 후 AP 소비로 방향 재조정 가능

**AP 비용 예시** (조종수 마크 0 기준):
- 회전 60° (1 스텝): 1 AP
- 회전 180° (반대 방향): 3 AP
- 포탑 회전: 회전 각도에 따라 0~2 AP (포탑 구동 파츠 의존)

## 7. 적 유형

### 7.1 적 카테고리

| 유형 | 특성 | 전술 대응 |
|---|---|---|
| **경량 기계** | 빠르고 약함 | 기총·소구경 속사포로 처리, 다수 등장 |
| **중장갑 기계** | 단단함, 측·후면 공략 필요 | 플랭킹 이동 후 대구경 관통 |
| **원거리 기계** | 사거리 우위 | 엄폐 이동 접근 또는 반격 사격 |
| **자폭 기계** | 근접 시 광역 자폭 | 기총 제압 사격 / 사거리 유지 |
| **지휘 기계** | 주변 적에 버프 | 우선 격파 (지휘권 소실 시 적 사기·정확도 하락) |
| **보스 기계** | 바이오 코어 탑재 | 패턴 공략, 스테이지 최종 전투 |

## 8. 전장 환경

### 8.1 지형 효과

| 지형 | 효과 |
|---|---|
| 아스팔트 | 기본 — 패널티 없음 |

- 건물 잔해, 바리케이드, 차량 잔해

## 9. 시네마틱 연출 방침

### 9.1 현재 방침 — 기본 사격 단일 연출

- **공격자측 엄폐 연출 폐기**: 기본 사격은 엄폐 유무와 무관하게 단일 "개활지 사격" 연출 사용
  - 엄폐 효과(명중률 보정·방호면 피해 차감)는 판정/UI로만 표현
  - 측면/전·후진/후진 전개 같은 복잡한 분기 연출은 도입하지 않음
- **대상측 엄폐 표시 유지**: 피격자의 `targetCoverFacets`를 hex edge 중점에 **월드 고정**으로 렌더
  - 공격 방향에 따른 회전/재배치 없음 — hex 방위 그대로
  - 전차 sortingOrder(hull=5/turret=6)보다 낮은 레이어(3)에 배치
- **방호면 3면 제한**(Small=1, Medium=3, Large=3)은 유지 — 게임플레이 데이터 레이어에서 강제
- **고정 포탑/바이오 코어**도 동일하게 기본 사격 연출 사용
- **MG 사격**은 2026-04-15 기준 주포와 동일한 프레이밍 구조로 통일 (`PlayOpenFireSequence` 재사용 + 스트림 볼리)

### 9.2 향후 확장 — 스킬 연계

공격자측 엄폐 전개 연출은 **스킬/승무원 스킬 시스템과 연계**하여 차별화된 연출로 재도입 예정:
- "차폐 돌파", "포탑 저격", "측면 강습" 등 고유 네이밍 스킬에 각각 전용 연출
- 기본 사격은 간소화 → 스킬 사용 시 시각적 임팩트가 부각됨
- 반응 사격(오버워치)은 현재 단일 연출 공유 — 스킬별 분화는 확장 범위
