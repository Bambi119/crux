# 6. 전투 시스템

## 전투 방식

**실시간 탑다운 서바이벌** — 뱀서류 기반이되, 전차 특성을 살린 전투

### 조작
- **WASD**: 차체 이동 (방향키 대응)
- **마우스**: 포탑 조준 (차체와 독립 회전)
- **좌클릭/자동**: 주포 사격
- **우클릭**: 보조 무기/스킬
- **Shift**: 긴급 기동 (엔진 부스트, 쿨다운 있음)

### 전투 흐름
1. 스테이지 시작 → 전차 배치
2. 웨이브 시작 → 적이 사방에서 밀려옴
3. 웨이브 클리어 → 짧은 휴식 (드랍 아이템 수집)
4. 다음 웨이브 → 적 강도 증가
5. 최종 웨이브/보스 → 클리어

## 관통/방호 시스템

실시간이지만 **관통 판정은 유지**:

```
적 공격 → 명중 → 입사각 계산 → 유효 장갑 비교 → 관통/도탄
```

- 도탄 시: 파편 파티클 + "땡!" 효과음
- 관통 시: 내부 데미지 + 모듈 손상 가능

### 유효 장갑 계산 (기존 유지)
```
유효 장갑 = 기본 장갑 / cos(입사각)
입사각 70도 이상 → 자동 도탄
```

### 모듈 피격
- **캐터필러**: 기동력 -50%
- **엔진**: 최대 속도 -30%, 과열 위험
- **포탑 구동장치**: 포탑 회전 속도 -50%
- **탄약고**: 유폭 판정 (HP 대량 손실 또는 즉사)
- 승무원 수리 스킬로 전투 중 복구 가능 (시간 소요)

## 적 유형

### 경량 기계
- 빠르고 약함, 대량 출현
- 뱀서의 일반 몹 역할

### 중장갑 기계
- 느리고 단단함, 관통이 어려움
- 측면/후면 공략 필요

### 원거리 기계
- 멀리서 사격, 회피 기동 필요

### 폭발 기계
- 근접 시 자폭, 광역 데미지

### 보스 기계
- 스테이지 최종 웨이브
- 패턴 공략 필요
- 바이오 코어 탑재

## 전장 환경

### 지형 효과
| 지형 | 효과 |
|------|------|
| 아스팔트 | 기본 |
| 진흙 | 이동 감속 |
| 잔해 | 파괴 가능, 시야 차단 |
| 물웅덩이 | 감속 + 엔진 과열 방지 |

### 파괴 가능 오브젝트
- 건물 잔해, 바리케이드, 차량 잔해
- 파괴 시 파편 파티클
- 일부는 적에게도 장애물 역할

## 전술 전투 모드 (턴제 그리드)

특정 조우(보스전·중요 스토리 교전)에서는 **턴제 전술 그리드 전투**로 전환.
실시간 서바이벌과 병존하며 조작·판정 체계가 다르다.

### 그리드 구조 — 육각형 (Hex)

- **타일 형태**: 정육각형 (pointy-top)
- **좌표계**: Axial coordinate `(q, r)` — Red Blob Games 표준
- **이웃 수**: 6방향 (60° 간격)
- **거리 함수**: 육각 거리 `(|dq| + |dr| + |dq+dr|) / 2`
- **이동 비용**: 1 셀당 정해진 AP — 모든 방향 동일 (대각선 파라독스 없음)

### 왜 육각인가

| 기준 | 근거 |
|------|------|
| 기하학적 정직성 | 모든 6방향이 진짜 등거리 — 사각+8방의 Chebyshev 왜곡 없음 |
| 이동 범위 | 원형에 가깝게 퍼짐 — 플레이어 직관과 일치 |
| 측면 공격 명료성 | 6구역 장갑이 타일 변 6개와 1:1 매칭 |
| 엄폐 일치성 | **6 방호면 = 6 타일 변** — 데이터·시각·물리 동일 언어 |
| 장르 정합성 | Panzer General·Steel Panthers 등 전차 전투 게임 역사적 표준 |
| 수학 성숙도 | Axial/Cube 좌표 체계가 업계 표준, 레퍼런스 풍부 |

### 엄폐 시스템 — 6면 방호 슬롯

- **엄폐물은 셀이 아닌 "변(edge)"** 에 속성 부여
- 각 셀은 최대 6개의 방호면 플래그를 가짐: `N / NE / SE / S / SW / NW`
- 공격자→대상 방향을 **6방향 슬롯에 스냅** 후, 해당 슬롯이 방호 플래그에 포함되면 엄폐 유효
- 엄폐율(`CoverRate`)은 HP에 비례
- **HP 감소 시 방호면 슬롯이 순차적으로 소실** (예: 완전 상태 3면 → HP 50% 2면 → HP 25% 1면)

```
예: 북쪽과 북동쪽을 막는 잔해 더미
protectedFacets = HexFacet.N | HexFacet.NE
→ 북에서 온 공격: 엄폐 유효
→ 북동에서 온 공격: 엄폐 유효
→ 동에서 온 공격: 엄폐 무효 (우회 성공)
```

**우회 전술성**: 대각 한 칸 이동만으로 방호면을 벗어날 수 있어, 이동 AP 소비로 엄폐 무효화가 가능 → 이동 결정이 명확한 트레이드오프를 만듦.

### 피격 구역 — 6섹터 장갑

전차 차체 각도(헐 방향) 기준 6등분:

| 구역 | 각도 범위 | 의미 | 일반 장갑 |
|------|-----------|------|-----------|
| **Front** | 0° ± 30° | 전면 | 최두꺼움 |
| **FrontRight** | 60° ± 30° | 우전 (측면 장갑의 강한 쪽) | 두꺼움 |
| **RearRight** | 120° ± 30° | 우후 | 얇음 |
| **Rear** | 180° ± 30° | 후면 | 가장 얇음 |
| **RearLeft** | 240° ± 30° | 좌후 | 얇음 |
| **FrontLeft** | 300° ± 30° | 좌전 (측면 장갑의 강한 쪽) | 두꺼움 |

**플랭킹 전술성**: 차체를 정면으로 돌리지 못한 상태에서 사격 받으면 즉시 측·후면 노출. 포탑 회전과 차체 회전을 독립적으로 관리해야 하는 이유.

### 유닛 회전 체계

- **헐 각도**: 60° 단위로 스냅 (6 방향)
- **포탑 각도**: 연속 각도 (차체와 독립 회전)
- 이동 시 차체가 이동 방향으로 자동 회전
- 정지 후 AP 소비로 방향 재조정 가능

### 턴 구조 (변경 없음)
- Player Turn → Enemy Turn 순환
- AP: 이동/사격/소화/연막
- 관통 판정, 모듈 피격, 화재/유폭 시스템은 그대로 유지
- 차이점은 "6면 판정" 으로 데이터 해상도만 변화

## 엄폐 사격 연출 (Cinematic) — 스코프 축소

### 현재 방침

- **공격자측 엄폐 연출 폐기**: 기본 사격은 엄폐 유무와 무관하게 단일 "개활지 사격" 연출 사용
  - 엄폐 효과(명중률 보정·방호면 피해 차감)는 판정/UI로만 표현
  - 측면/전·후진/후진 전개 같은 복잡한 분기 연출은 도입하지 않음
- **대상측 엄폐 표시 유지**: 피격자의 `targetCoverFacets`를 hex edge 중점에 **월드 고정**으로 렌더
  - 공격 방향에 따른 회전/재배치 없음 — hex 방위 그대로
  - 전차 sortingOrder(hull=5/turret=6)보다 낮은 레이어(3)에 배치
- **방호면 3면 제한** (Small=1, Medium=3, Large=3)은 유지 — 게임플레이 데이터 레이어에서 강제
- **고정 포탑/바이오 코어**도 동일하게 기본 사격 연출 사용

### 향후 확장 — 스킬 연계

공격자측 엄폐 전개 연출은 **스킬/승무원 스킬 시스템과 연계**하여 차별화된 연출로 재도입 예정:
- "차폐 돌파", "포탑 저격", "측면 강습" 등 고유 네이밍 스킬에 각각 전용 연출
- 기본 사격은 간소화 → 스킬 사용 시 시각적 임팩트가 부각됨
