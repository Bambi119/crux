# CRUX (철갑을 두른 당나귀) — 기획서 v3

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 2D 턴제 전차 전략 + 전투 연출 |
| 엔진 | Unity 6.3 LTS |
| 시점 | 탑다운 2D |
| 세계관 | 포스트 아포칼립스. 외계 바이오 코어에 감염된 기계들과 싸우는 소년 아스트라와 구형 전차 로시난테 |
| 레퍼런스 | 어드밴스드 대전략, 아머드코어 초기작, XCOM, Men of War |

---

## 2. 게임 흐름

```
[전략맵 씬 — StrategyScene]
  유닛 선택 → AP 이동(방향 선택 포함) → 적 선택 → 무기/탄종 선택
  → 명중률/관통률 확인 → "사격" 확정 → 결과 미리 계산
      ↓ SceneManager.LoadScene("FireActionScene")

[전투 연출 씬 — FireActionScene]
  카메라 줌인 → 전차 전진 → 포탑 회전 → 포연 + 포탄 비행
  → 착탄 즉시 판정 + 이펙트 (도탄 튕김 / 피격 충격 / 관통 폭발)
  → 결과 표시 → 클릭하여 계속
      ↓ SceneManager.LoadScene("StrategyScene")

[전략맵 복귀]
  상태 복원 → 데미지 적용 → 다음 유닛/적 턴 진행
```

## 5. 턴 시스템

### 5.1 턴 페이즈
```
PlayerTurn → EnemyTurn → (Cinematic) → 반복
```

### 5.2 AP (Action Point) 체계

| 유닛 | 최대 AP |
|------|---------|
| 플레이어 (로시난테) | 6 |
| 경량 드론 | 5 |
| 중장갑 전차 | 4 |

| 행동 | AP 비용 |
|------|---------|
| 이동 (셀당) | 1 |
| 주포 사격 | 3 |
| 경계 모드 | 2 (미구현) |

### 5.3 플레이어 입력

| 키 | 기능 |
|----|------|
| M | 이동 모드 진입 |
| F | 사격 모드 진입 |
| Space | 턴 종료 |
| ESC | 모드 취소 |
| 1/2/3 | 무기 선택 (주포/동축기총/탑재기총) |
| ↑↓←→ / WASD | 이동 후 방향 선택 |
| Enter / 좌클릭 | 이동 확정 |

### 5.4 이동 + 방향 선택
1. M키로 이동 모드 → 이동 범위 하이라이트
2. 셀 클릭 → 목적지 선택 + 방향 선택 모드 진입
3. 화살표키로 최종 방향 설정 (8방향)
4. AP 비용 표시
5. Enter/클릭으로 확정 → 이동 애니메이션 → 도착 후 회전

## 6. 유닛 시스템

### 6.1 유닛 데이터 (TankDataSO)

| 필드 | 설명 |
|------|------|
| tankName | 유닛 이름 |
| maxAP | 최대 행동력 |
| moveSpeed | 시각적 이동 속도 |
| armor | 장갑 프로필 (전면/측면/후면/포탑) |
| turretRotationSpeed | 포탑 회전 속도 |
| fireCost | 사격 AP 비용 |
| maxHP | 최대 체력 |
| muzzleOffset | 포구 위치 오프셋 (Inspector 조절 가능) |
| caterpillarHP | 캐터필러 내구력 |
| engineHP | 엔진 내구력 |
| ammoRackHP | 탄약고 내구력 |

### 6.2 장갑 프로필 (ArmorProfile)

| 부위 | 설명 |
|------|------|
| front | 전면 장갑 (mm) |
| side | 측면 장갑 (mm) |
| rear | 후면 장갑 (mm) |
| turret | 포탑 장갑 (mm) |

### 6.3 모듈 손상
- 관통 시 20% 확률로 모듈 랜덤 손상 (데미지의 50%)
- 캐터필러 파손: 이동 불가
- 엔진 파손: 이동 불가
- 탄약고 파손: **즉시 파괴** (HP = 0)

### 6.4 방향 시스템
- 나침반 방식: **0° = 북, 90° = 동, 180° = 남, 270° = 서** (시계방향)
- `AngleUtil`: 나침반 ↔ Unity Euler Z ↔ 방향 벡터 변환
- 스프라이트 기본 방향: → (오른쪽) 통일
  - 적 스프라이트(↓)는 SpriteContainer에서 -90° 보정
- 포탑은 독립 회전, pivot 기반 회전 중심

## 7. 전투 판정 시스템

### 7.1 명중 판정
```
명중률 = 기본(85%) - 거리당 감소(4%/셀) - 엄폐 감소(25%)
```
- 엄폐 감소는 대상의 엄폐율 기반

### 7.2 엄폐물 흡수 판정
- 대상이 엄폐물 인접 시, 엄폐율만큼 확률로 엄폐물이 대신 피격
- 엄폐율 = `최대 엄폐율 × (현재HP / 최대HP)`
- 엄폐물 피격 시 HP 감소, 파괴 시 셀 상태 변경 + 인접 셀 엄폐 해제

### 7.3 피격 부위 판별
- 공격 방향과 대상 차체 방향의 각도 차로 판별
  - 0~45°: **정면**
  - 45~135°: **측면**
  - 135~180°: **후면**

### 7.4 입사각 계산
- 포탄 방향과 해당 부위 장갑면 법선의 내적으로 실제 입사각 계산
- 부위별 법선: 정면=전방, 후면=후방, 측면=우측

### 7.5 유효 장갑
```
유효 장갑 = 기본 장갑 / cos(입사각)
```
- 입사각 0° → 기본 장갑 그대로
- 입사각 45° → × 1.41배
- 입사각 60° → × 2배
- **입사각 70° 이상 → 자동 도탄**

### 7.6 관통 판정 (3단계)
```
비율 = 관통력 / 유효 장갑

비율 > 1.2  → 관통 (250% 데미지)
비율 0.8~1.2 → 피격 (100% 데미지, 경계값 근처 확률적 관통)
비율 < 0.8  → 도탄 (3% 데미지)
```

## 9. 전투 연출 씬

### 9.1 연출 분기

| 조건 | 연출 |
|------|------|
| 공격자 엄폐 중 | 엄폐물 뒤에서 전진 → 빠져나옴 → 포탑 회전 → 사격 |
| 공격자 개활지 | 차체 방향으로 전진 → 정차 → 포탑 회전 → 사격 |
| 기관총 | 방향 맞춤 → 버스트 연사 (개별 탄착 + 즉시 판정) |

### 9.2 주포 연출 시퀀스
```
[전진] 차체 방향으로 이동 (0.5~0.6초)
[정차] 정지! (0.2초)
[조준] 포탑이 적 방향으로 회전 (0.35~0.4초)
[사격] 포연(MuzzleFlash) + 카메라 쉐이크
[비행] 포탄 직선 이동 (0.15초)
[착탄] 즉시 이펙트 + 판정 + 데미지 팝업
```

### 9.3 착탄 판정별 연출

| 판정 | 연출 |
|------|------|
| 빗나감 (Miss) | 회색 텍스트 "빗나감!" (0.3초) |
| 도탄 (Ricochet) | 스파크 이펙트 → 포탄이 랜덤 방향으로 빠르게 튕겨나감 (0.15초) → 지면 충돌 |
| 피격 (Hit) | 충격 + 반동 + 카메라 쉐이크 → 연기 |
| 관통 (Penetration) | 반동 → 포탄 관통 출구 화염 → 내부 폭발 + 흑연 |

### 9.4 데미지 팝업 (DamagePopup)
- 착탄 지점에서 위로 떠오르며 페이드아웃
- 색상: 회색(빗나감), 황색(도탄), 주황색(피격), 빨간색(관통)
- 기관총: SpawnSmall()로 소형 팝업

### 9.5 포구 위치 (MuzzleOffset)
- `TankDataSO.muzzleOffset`로 Inspector에서 조정 가능
- 포탑 회전 방향에 맞춰 자동 변환

## 11. 적 AI

### 11.1 행동 우선순위
1. 플레이어를 향해 방향 전환 (`FaceToward`)
2. 사거리 내이면 → **사격** (연출 씬 전환)
3. 사거리 밖이면 → 플레이어 방향으로 **이동** → 이동 후 사격 가능하면 사격

### 11.2 턴 중 씬 전환 처리
- 적 사격 시 `currentEnemyIndex` 기록 후 연출 씬 전환
- 복귀 후 `nextEnemyIndex`부터 나머지 적 행동 재개
- 모든 적 행동 완료 후 플레이어 턴으로 전환

## 12. 씬 전환 및 상태 보존

### 12.1 데이터 전달 (FireActionContext)
- `FireActionData` 구조체: 공격자/대상 정보, 스프라이트, 무기, 판정 결과
- static 클래스로 씬 간 유지

### 12.2 상태 저장 (BattleStateStorage)
- 사격 직전 `SaveBattleState()` 호출
- 저장 내용: 모든 유닛의 위치/HP/AP/방향, 턴 수, 현재 페이즈, 엄폐물 HP, 다음 적 인덱스

### 12.3 복원 흐름
```
StrategyScene Start()
  → FireActionContext.HasPendingAction 확인
  → ApplyPendingResult()
    → InitializeBattle() (맵/유닛 생성)
    → RestoreState() (저장된 위치/HP/AP 복원)
    → 데미지 적용 (주포 or 기관총 결과)
    → 적 턴이었으면 ProcessEnemyTurn(nextEnemyIndex) 재개
```

## 13. 스프라이트 시스템

### 13.1 스프라이트 방향 통일
- 모든 유닛 스프라이트는 **→ (오른쪽)** 기준
- 적 스프라이트(↓)는 `SpriteContainer`로 감싸서 -90° 보정
- 배틀씬에서도 `spriteRotOffset`을 `FireActionData`로 전달하여 동일 구조 적용

### 13.2 포탑 구조
- 포탑은 차체의 자식 오브젝트 (독립 회전 가능)
- `localPosition = (0, 0, 0)` — 스프라이트 pivot으로 회전 중심 설정
- Sprite Editor에서 pivot을 포탑 링 중심으로 조정

### 13.3 절차적 스프라이트 (TankSpriteGenerator)
- 스프라이트 에셋이 없을 경우 코드로 생성
- static 캐싱으로 Resource ID 오버플로 방지

### 13.4 AI 생성 스프라이트
- Gemini로 생성, 수동 알파 처리
- Import: Sprite Mode Single, Filter Point, Compression None, PPU 256
