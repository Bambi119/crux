## 운영 원칙
1. **기존 모듈 우선** — 새로 만들기 전에 기존 시스템·유틸리티부터 확인
2. **자가 수정** — 도구 실패 시 원인 파악 → 수정 → 테스트 → 문서 갱신
3. **문서 선행** — 대규모 코드 변경 전 설계 문서 먼저 갱신
4. **게임 데이터 보호** — 세이브 파일, 밸런스 테이블, 에셋 원본은 파괴적 수정 금지
